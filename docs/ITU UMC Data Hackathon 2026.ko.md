# 문제 진술

#### 인프라 접근성은 이미 높지만 차원별로 의미 있는 이용 수준은 여전히 다를 수 있는 서울과 같은 초연결 도시에서 UMC(Universal and Meaningful Connectivity)는 어떻게 측정되어야 하는가?

#### 특정 지역, 집단, 또는 UMC 차원이 의미 있는 연결성 격차에 더 노출되어 있는가? 그리고 이는 장소에 민감하면서도 사람을 중심에 두는 정책 설계에 어떠한 함의를 갖는가?

서울은 흔히 완전히 연결된 도시로 간주되지만, 높은 네트워크 가용성만으로는 시민이 디지털 서비스를 의미 있고 안전하며 부담 가능하고 역량 있는 방식으로 이용할 수 있는지를 그 자체로 보여주지는 못한다. 따라서 본 연구는 UMC를 단순한 인프라 점검이 아니라 측정의 문제로 설정한다.

이러한 질문에 답하기 위해 본 보고서는 자치구 단위의 UMC 지수 구축, 개인의 디지털 이용에 대한 위계 선형 모형, 그리고 구조화된 플랫폼 텍스트 및 Bayesian 신호 분석을 연결한다. 그 목적은 특정 자치구를 영구적인 결핍 지역으로 낙인찍는 것이 아니라, 행정 지표·개인 취약성·생활세계 신호를 함께 읽어야 할 지점이 어디인지를 식별하는 데 있다.

# 핵심 결과 요약

아래의 결과는 본 보고서의 두 가지 연구 질문, 즉 초연결 도시에서 UMC를 어떻게 측정해야 하는가, 그리고 특정 지역·집단·차원이 의미 있는 연결성 격차에 더 노출되어 있는가에 대한 답이다. 서울에서 의미 있는 연결성은 단순한 네트워크 가용성의 문제가 아니다. 증거는 자치구 조건, 개인의 취약성, 그리고 플랫폼에서 가시화되는 서비스 이용 경로상의 문제들이 각기 동일한 측정 문제의 서로 다른 측면을 드러내는, 층화된 도시 격차를 가리킨다.

#### 1. 자치구 단위의 UMC 격차는 단순한 인프라의 문제가 아니라 다차원적이다.

자치구 지수는 서울의 연결성 격차가 차원별로 다르게 나타남을 보여준다. 종합 점수는 서울 내부 비교 지수로, 각 지표는 서울 자치구 분포 범위에 대해 정규화되고 UMC 6차원이 평균되므로, 그 값은 UMC 달성의 절대 수준이 아니라 자치구의 상대적 위치를 나타낸다. 점수가 낮은 자치구라도 약점이 동일한 방식으로 나타나지는 않는다. 어떤 격차는 기본적인 물리적 연결보다는 Devices, Safety, Affordability, Digital Skills와 결부되어 있다. 따라서 Digital Desert 라벨은 자치구 전체에 대한 고정된 진단이 아니라, 잠정적인 타겟팅 신호로 사용된다.

#### 2. 디지털 이용 격차는 광범위한 자치구 조건보다 개인 취약성이 더 많이 설명한다.

다층 분석은 디지털 이용의 차이가 주로 자치구 내부에서, 즉 학력·연령·가구 형태·소득·직업·주거 조건이 서로 다른 거주자들 사이에서 구조화되어 있음을 시사한다. 낮은 학력, 고령, 1인 가구, 그리고 연령과 학력의 결합으로 인한 취약성이 가장 뚜렷한 실질적 패턴이다. 약 99.5%라는 수치는 HLM 분산 분해 후 디지털 이용 점수의 분산 가운데 자치구 내부, 즉 개인 간 수준에서 위치한 비중을 가리킨다. 이는 모형의 정확도나 연결된 시민의 비율이 아니다. 자치구 지표는 맥락적 신호로서 여전히 유용하지만, 개인의 디지털 이용을 직접 일으키는 인과적 요인으로 읽혀서는 안 된다.

#### 3. 플랫폼에서 가시화되는 신호는 행정·서베이 자료가 충분히 포착하지 못하는 경로를 설명하는 데 도움이 된다.

이 결과는 분석 3, 즉 구조화된 플랫폼 텍스트 분류와 EB 수축 율 모형을 활용한 측정된 UMC prior와 플랫폼 가시 신호 사이의 탐색적 비교에 근거한다. 모형은 작성 시점 시간대 생활인구 노출을 분모로 사용하며, per-100k person-years 단위와 차원 내 표준화 z_shift로 어긋남을 보고한다. 이 층위들은 신호가 어긋나는 지점, 특히 Devices와 Digital Skills 주변의 분기점을 식별한다. 이는 탐색적 증거 통합 신호이며, 유병률을 추정하지 않고 행정 지표가 틀렸음을 증명하지도 않는다. 그 가치는 시민이 필요한 순간에 정보 부재, 취약한 지원 경로, 역량 제약, 또는 신뢰·안전에 대한 우려에 직면할 수 있는 지점을 보여주는 데 있다.

#### 정책 함의

이로부터 도출되는 정책 논리는 순차적이다. 먼저 취약한 개인과 가구를 우선시하고, 자치구 UMC 프로파일을 활용해 아웃리치를 표적화하고 차원별로 개입을 맞춤화하며, 단일 인프라 중심 사업에 의존하기보다 역량·안전·부담 가능성·접근 지원을 결합한다.

# 1. 개관

## 1.1 이론적 이해

본 프로젝트는 서울의 디지털 연결성을 세 가지 분석 단계로 검토하고, 그 결과에 기반해 정책 권고를 도출한다. 디지털 연결성은 ITU가 제안한 Universal Meaningful Connectivity(UMC) 프레임워크에 근거한다. 본 프로젝트의 분석들은 분리된 산출물들의 병렬이 아니라, 하나의 메타이론적 틀 안에서 해석된다. 메타이론은 현실의 본질에 대한 종합적 관점으로서 연구와 실천에 철학적 토대를 제공하며(Allana & Clark, 2018), 각 방법론의 존재론적 위치를 명시할 근거를 제공한다. 관찰된 현상이 서로 다른 존재론적 수준에서 개념으로 일반화되는 과정을 명시적으로 보여줌으로써, 본 프로젝트는 자체의 타당성을 확보하고자 한다. 이러한 접근은 프로젝트 자체의 타당성을 제공할 뿐 아니라, 다른 맥락에 적용할 때의 명확한 근거 또한 제공한다는 장점을 갖는다.

#### Evidence Transfer and Local Conditions

이는 현재의 정책 집행에 대한 대안적 참조점을 제공할 수 있다. 증거 기반 정책이 보편화되면서, 원래 맥락의 제도적·문화적 조건을 충분히 고려하지 않은 채 분석 결과가 차용·적용되는 사례가 증가해 왔다(Dolowitz & Marsh, 2000). 동시에 증거 기반 패러다임 자체에 대한 비판도 누적되어 왔다. 대표적인 비판은 좁은 의미의 증거 기반 틀이 정책 결정의 복합적·맥락 의존적·가치 함축적 성격을 충분히 포착하지 못한다는 것이다(Greenhalgh & Russell, 2009). 이에 따라 증거 기반(evidence-based) 정책에서 증거 정보(evidence-informed) 정책으로의 전환이 논의되어 왔으며(Head, 2010), 증거의 역할은 정책의 체계적 기반을 구성하는 것이 아니라 정책을 안내하는 것으로 재정의되고 있다. 본 프로젝트는 경험적 증거의 중요성을 축소하는 것이 아니라, 이론적 틀에 기반해 증거를 다른 맥락에 적용하기 위해 요구되는 조건과 핵심 구조를 상세히 명확히 하는 것을 목표로 한다. 각 분석의 역할과 그 상호 연결, 그리고 정책 권고로 이어지는 해석 구조를 명시적으로 제시하면서, 해석에 필요한 핵심 맥락 정보 또한 함께 제시한다. 이는 개별 분석 방법과 구분되는, 프로젝트 자체의 사례 연구적 성격을 반영한다.

#### Three Analytical Layers

개별 분석 방법론은 세 가지 유형으로 구분된다. 첫째, UMC의 이론적 틀을 정책 집행의 주체인 서울 25개 자치구에 적용해 상대적 격차를 측정한다. 다음으로, 다층 모형을 활용해 개인의 디지털 연결성 문제에 대한 개인 요인과 자치구 수준 맥락 요인의 기여를 분리한다. 마지막으로, 앞선 분석에서 일반화된 서울의 디지털 연결성 문제에 관한 플랫폼 가시 증거를 비정형 텍스트를 통해 관찰된 개인 현상에 다시 연결한다. 여기에는 UMC의 개념적 틀에 따라 개인의 경험을 추상화하고 재구성하는 절차가 포함된다. ITU가 제안한 UMC 개념을 집계 결과 및 플랫폼 가시 생활세계 신호와 비교한다. 마지막으로, 실제 현상이 발생할 수 있는 그럴듯한 경로를 그 개념으로부터 해석한다. 이러한 수준 간 이행은 비판적 실재론이 제안한 층화된 존재론, 즉 경험적·실재적·실제적 영역의 구분에 근거한다(Bhaskar, 1975).

이러한 절차의 필요성은 가상의 상황을 통해 설명할 수 있다. 예를 들어, 급행 지하철 노선이 정차하는 역세권 주거지가 그렇지 않은 역세권 주거지보다 주택 가격이 높다는 경험적 증거가 관찰되었다고 가정하자. 결과의 타당성과는 별개로, 그 발견 자체를 의미 있는 지식으로 간주하기는 어렵다. 그것은 서울에만 적용될 수 있다. 또한 이 결과에 근거해 모든 역을 급행 정차역으로 만드는 것은 정치적으로도 논리적으로도 가능하지 않다. 앞의 상황에서 한 가지 예시는, 서울의 높은 시간 가치가 통행 시간을 단축시키는 급행 노선이 주택 가격에 반영되도록 하는 메커니즘을 식별하는 것이다. 다음 절에서는 개별 분석 방법론 자체가 다른 맥락에 적용될 때 갖는 이점을 설명한다.

## 1.2 분석 방법의 이해

본 절의 목적은 세 가지 분석 방법을 설명하고, 위에서 제시한 메타이론적 틀과는 별개로 각 방법론 자체가 다른 맥락에 적용될 수 있는 근거를 명확히 하는 데 있다. 첫째, 자료에 대한 회귀 기반 분석을 통해 경험적 패턴을 식별한다. 논의에서는 개별 관찰을 측정·추상화한 뒤 다시 개인 현상으로 복원하면서 UMC 개념이 경험적으로도 발현되는지를 점검하는 메타 수준 구조의 타당성과, 개별 방법론의 적용 가능성을 분리해 다룬다.

#### Regional Disparities in UMC in Seoul

먼저, 서울의 자치구 단위 UMC를 측정해 격차가 어디에, 어떤 차원에서 존재하는지를 보인다. 측정 지표는 ITU(2023)가 제안한 6차원의 개념적 정의에 기반하되, 서울의 자료 환경과 도시 특성에 맞게 조작화한다. 정책을 실제로 집행하는 주체는 국가 수준뿐 아니라 지방정부를 포함한 다양한 단위에 분포한다. 이에 따라 본 프로젝트는 서울 25개 자치구를 측정 및 분석의 단위로 설정하여, 분석 단위가 정책 집행 단위와 일치하도록 한다. 개념적 정의와 연구 지역에 적합한 조작적 정의가 분리 적용되므로, 동일한 6차원 구조를 유지하면서도 각 도시의 자료 환경에 맞게 지표를 대체할 수 있다. 측정 지표의 선정과 결과는 3.1절에서 상세히 다룬다.

#### Individual and District Characteristics and Digital Connectivity

다음으로, 중첩 자료에서 개인 수준 요인과 자치구 수준 요인을 분리하는 모형인 2수준 위계 선형 모형(HLM)을 활용해, 어떤 집단에서 그리고 어떤 맥락 조건에서 디지털 이용의 차이가 가장 두드러지는지를 검토한다. 그 목적은 인과 추론이 아니라 다층 연관성 분석을 수행하는 데 있다. 동일한 행정구역 안에서도 학력·연령·가구 형태와 같은 개인 특성에 따라 디지털 이용은 상당히 달라질 수 있다. HLM은 개인이 자치구에 중첩되어 있는 위계적 자료 구조를 설명하기 위한 표준적 방법이며, 동일한 분석 틀은 해당 맥락의 인구학적 특성과 인프라 조건에 따라 투입 변수를 교체함으로써 다른 도시에도 적용될 수 있다. 변수 선정, 모형 설정, 그리고 교차 수준 상호작용의 세부 설계는 3.2절에서 제시한다.

#### Multi-Agent Qualitative Research

마지막으로, 앞선 두 분석이 포착하지 못하는 플랫폼 가시 생활세계 신호를 비정형 텍스트로부터 식별하고 이를 UMC 틀과 관련지어 해석한다. 개별 사례는 곧바로 자치구 단위의 UMC 개념과 동일시되지 않는다. 오히려 구조화된 코딩과 해석의 절차를 거쳐 앞선 관찰 결과와 비교됨으로써, 가능한 경험 경로를 예시한다. 이 절차는 다층 모형을 증명하거나 모집단 유병률을 추정하지 않는다. 그 가치는 행정 자료와 서베이 기반 패턴이 일상적 상황에서 어떻게 나타날 수 있는지를 가시화하는 데 있다.

텍스트 분석은 두 가지 요건을 갖는다. 첫째, 개별 텍스트 사례가 디지털 연결성과 관련 있는지를 판단하고, 사전에 정의된 UMC 6차원의 이론적 틀에 따라 분류할 수 있어야 한다. 둘째, 분석 방법이 특정 언어에 강하게 묶여 있지 않아야 한다. UMC 틀이 포괄적이므로 텍스트 분석 역시 언어 독립성을 요구한다. 이 두 조건을 충족하기 위해, 본 연구는 사전에 정의된 기준에 따라 플랫폼 텍스트를 분류하는 구조화된 코딩 보조도구로 사용된 LLM(대규모 언어 모형)을 이론 주도적 텍스트 분류 도구로 적용한다. LLM은 형태소 수준이 아니라 맥락 수준에서 의미를 처리하므로 한국어 일상 텍스트의 비표준 표현에 대해 견고하며, 연구자는 UMC 차원을 프롬프트에 직접 제시함으로써 분류 기준을 통제할 수 있다. 동일한 프롬프트 구조가 다른 언어의 텍스트에도 적용될 수 있다는 점에서 방법론적 이전 가능성이 확보되지만, 이는 인간 검증을 대체하지는 않는다.

요약하면, 본 프로젝트의 구조 안에서 2장은 3장의 결과를 읽기 위한 해석 맥락을 제공하며, 3.3절은 3.1절과 3.2절의 결과를 생활세계의 경험과 다시 연결하는 매개 단계로 기능한다. 세 가지 분석은 서로 다른 질문에 답하는 동시에, 한 분석의 산출이 다음 분석의 투입이 되는 순차적 구조를 이룬다(Figure 1 참조). 개념 구조와 조작적 정의를 분리하고 어떤 요소가 대체 가능하며 어떤 요소가 고정되어 있는지를 명시함으로써, 각 방법론은 동일한 분석 틀이 서울을 넘어 적용될 수 있는 조건을 확립한다(Figure 1).

# 2. 이론적 배경

본 장은 분석을 위한 이론적 토대와 3장의 해석 맥락을 제시한다. 먼저 디지털 연결성에 영향을 미치는 개인 및 지역 요인을 검토한다. 이어 공공정책에서 사람 기반 접근과 장소 기반 접근의 구분, 그리고 최근의 장소 민감 접근(place-sensitive approach) 논의의 배경을 검토한다. 다음으로 본 프로젝트의 메타이론적 틀인 비판적 실재론과 이를 적용한 경험 연구 사례를 검토한다. 마지막으로 분석 결과 해석에 필요한 맥락 정보를 제공하기 위해 서울의 인구·행정·인프라 조건을 살펴본다.

## 2.1 보편적·의미 있는 연결성 (UMC)

Universal Meaningful Connectivity(UMC)는 모든 사람이 부담 가능한 비용으로 안전하고 만족스럽고 풍요로우며 생산적인 방식으로 인터넷을 이용할 수 있어야 한다는 정책 목표이다(ITU, 2025). 본 연구는 이러한 목표를 비판적 실재론적 관점에서 두 가지 개념적 갈래를 검토하며 발전시킨다. 첫째는 UMC 지표의 역량 기반적 성격에 관한 것이고, 둘째는 집합적 UMC와 개인의 디지털 연결성 간의 간극에 관한 것이다.

첫 번째 갈래는 Sen의 역량 접근에서 출발한다. Sen(1985)은 안녕(well-being)은 사람들이 그 자체로 보유한 자원이 아니라, 실제로 무엇을 할 수 있고 무엇이 될 수 있는지에 의해 평가되어야 한다고 주장했다. 이후의 역량 연구는 동일한 자원이 개인 조건, 사회적 제도, 제도적 환경에 따라 서로 다른 실제 기회로 전환될 수 있음을 강조해 왔다(Robeyns, 2005). 이러한 논리는 ITU의 보편적이고 의미 있는 연결성 개념과 직접 연결된다. 단순한 네트워크 접근이나 단말 보유만으로는 충분하지 않으며, 사람들이 그 자원을 일상생활에서 의미 있는 이용으로 전환할 수 있어야 한다. 역량 기반 접근은 또한 임계치(threshold)의 문제, 즉 사회적으로 수용 가능한 최소 수준이 무엇인지의 문제를 제기한다. 연결성이 낮은 환경에서 이 임계치는 흔히 기본적인 인터넷 접근의 유무로 일차적으로 표현된다.

그러나 서울은 이미 고도화된 디지털 인프라와 높은 수준의 스마트폰 보급률을 보유하고 있으므로, 임계치 개념의 다른 조작화를 요구한다. 본래의 UMC 틀에서 임계치 기반 측정은 의미 있는 연결성의 사회적으로 수용 가능한 최소 수준이 달성되었는지를 식별하는 데 적합하다. 서울에서는 많은 기본 접근 임계치가 이미 충족되어 있으므로, 임계치 기준만을 적용한다면 자치구 간 의미 있는 차이를 관찰하기 어렵다. 따라서 본 연구의 경험적 전략은 임계치 접근의 규범적 관심은 유지하되, 25개 자치구의 상대적 위치를 공통 척도에서 비교하기 위해 서울 내부 Min-Max 정규화를 통해 이를 조작화한다. 그 결과 임계치의 문제는 기본 접근의 유무에서, 일상적 도시 생활 속 디지털 연결의 상대적 품질·부담 가능성·이용 용이성·안전성으로 이동한다.

두 번째 갈래는 국가 또는 지역 수준의 UMC와 개인 수준의 디지털 연결성 사이의 개념적 간극에 관한 것이다. UMC는 본래 국가 수준의 정책 및 측정 개념으로 정식화되었으며, ICT Development Index는 보편적이고 의미 있는 연결성이 집계 수준에서 어느 정도인지를 평가한다(International Telecommunication Union, 2025). 그러나 개인의 디지털 연결성은 국가 또는 자치구 수준 지표의 단순 산술합으로 환원될 수 없다. 디지털 불평등 연구는 물리적 접근이 확대된 이후에도 기술, 이용 형태, 결과, 사회적 위치에서의 차이가 지속됨을 보여 준다(Hargittai, 2002; Helsper, 2012; van Deursen & van Dijk, 2015; Scheerder et al., 2017). 즉, 동일한 지역 조건 아래에서도 사람들의 디지털 경험은 서로 다를 수 있다.

이 간극은 본 연구의 다층적 설계를 위한 개념적 근거를 제공한다. 본 연구가 비판적 실재론을 채택하는 이유는 자치구 지표, 개인 서베이 응답, 플랫폼 게시물이 단지 서로 다른 척도에서 동일한 현상을 측정하는 것이 아니기 때문이다. 오히려 그것들은 실재의 서로 다른 층위, 즉 구조적 조건, 개인 이용, 생활 사건에 대응한다. 분석 후반부에서 사용되는 Bayesian 갱신은 한 층위를 다른 층위로 환원하지 않으면서 이 층위들을 연결할 통제되었으나 탐색적인 방법을 제공한다. 자치구 단위의 UMC 점수는 맥락적 조건에 관한 사전 정보를 제공하고, 분류된 플랫폼 게시물은 결핍이 어떻게 서술되고 가시화되는지에 관한 플랫폼 가시 신호를 제공한다. 따라서 사후의 변화는 불만 유병률의 증가나 지역 결핍의 확증으로 해석되지 않으며, 집합적 UMC 조건과 관찰된 생활 경험 사이의 관계를 검토하는 데 도움을 주는 신호로 해석된다.

## 2.2 정책

공공정책의 설계에서, 개인의 특성을 표적으로 하는 개입, 즉 사람 기반 접근과 장소 조건을 표적으로 하는 개입, 즉 장소 기반 접근의 구분은 오랜 논쟁이다. 사람 기반 정책은 개인 역량, 소득, 교육과 같은 인적 특성을 직접 지원함으로써 결과를 개선하고자 한다. 장소 기반 정책은 특정 지역의 인프라, 제도, 환경 조건을 개선함으로써 그 지역 거주자의 결과를 간접적으로 개선하고자 한다(Neumark & Simpson, 2015). 두 접근은 상호 배타적이지 않지만, 정책 자원의 배분과 개입 경로가 다르기 때문에 정책 집행의 관점에서 별도로 설계될 필요가 있다.

### 2.2.1 디지털 격차에 영향을 미치는 요인: 개인 특성과 지역 조건

디지털 격차 문헌에서, 개인 특성에 주목하는 연구들은 동일한 지역 내에서도 개인 특성이 디지털 이용 격차를 설명하는 주된 요인임을 보여 준다. Hargittai(2010)는 인터넷 접근 이후의 격차, 즉 이용 패턴의 차이가 실질적 불평등을 결정한다고 주장했다. 이 결과는 사회경제적 지위에 따라 디지털 이용 격차가 지속될 수 있음을 시사한다. Chipeva et al.(2018)은 다양한 국가에 걸쳐 유용성(usefulness)이 기술 이용의 가장 강력한 동기임을 보여주는 한편, 개인의 내적 특성이 디지털 기기 이용에 유의하게 영향을 미친다는 점도 보였다. Borges et al.(2020)은 앙골라 사례를 통해 ICT 기술에 대한 습관과 숙련도가 단말 보급 이전 단계의 격차에서 핵심 요인임을 보고했다. 이러한 특성 가운데 연령과 학력은 불리함과 가장 일관되게 결부되는 특성으로 반복해서 나타난다. 중국의 고령층 디지털 격차(gray digital divide)를 연구한 Li & Kostka(2024)는 고령층이 디지털 기술을 학습하고 이용하는 데에서 다중적 장벽에 직면하며 적응을 위해 사회적 지원에 의존한다는 점, 그리고 그 태도가 조심스러운 낙관에서 뒤처졌다는 인식에 이르기까지 폭넓게 나타난다는 점을 발견했다. 본 연구의 맥락에 보다 가까운 급속한 고령화 사회인 한국을 대상으로, Jung et al.(2025)은 고령층의 디지털 리터러시가 삶의 만족도와 정(+)의 연관을 보이며, 그 일부가 디지털 참여를 통해 형성되는 사회적 자본을 매개로 한다는 점을 보였다. 학력과 직업 숙련도 역시 이와 평행한 격차를 이룬다. 2022년 헝가리 센서스를 활용한 Számel et al.(2025)은 가장 기초적인 접근 격차가 저숙련 집단에서 가장 지속적으로 남아 있으며, 사회계급이 디지털 기술과 이용 형태를 모두 계속 좌우함을 발견했다. 접근과 역량이 갖추어진 경우에도, Eynon(2023)은 낮은 사회경제적 배경을 가진 이용자가 디지털 역량을 갖추었더라도 온라인에서 할 수 있는 일이 그들의 폭넓은 여건에 의해 제약됨을 보였다. Hilbert(2011)의 18개 라틴아메리카 국가 분석은 성별 디지털 격차가 존재하며 성평등 수준이 높은 국가에서 인터넷 이용이 더 활발함을 더하여 보여 주는데, 이는 여러 개인 수준 축이 동시에 작동하며 저학력 고령층이 불리함이 반복적으로 집중되는 지점임을 드러낸다.

동시에, 개인 특성을 통제한 이후에도 지역 환경이 디지털 이용에 대한 추가적 설명력을 갖는다는 증거도 누적되어 왔다. 미국 자료에 HLM을 적용한 Mossberger et al.(2006)은 인터넷 접근 격차가 인종 그 자체보다 고빈곤 근린 환경과 연관됨을 보였다. 이는 지역의 구조적 조건이 개인의 결과가 산출되는 맥락을 형성할 수 있음을 시사한다. 영국의 지역 디지털 포용 전략을 비교한 질적 연구에서 Branstrator-Erickson & Ellison(2023)은 지역 디지털 포용 정책이 개인 수준 요인만으로 설명될 수 없으며, 지방정부, 시민 조직, 교육 기관, 디지털 서비스 제공자 간 조직 간 거버넌스 구조에 의해 추가로 형성됨을 주장했다.

### 2.2.2 정책 구분: 사람 기반, 장소 기반, 장소 민감형 접근

앞 절에서 검토한 두 가지 유형의 요인은 공공정책 설계에서의 사람 기반 접근과 장소 기반 접근의 구분에 대응한다. 사람 기반 정책은 개인 역량, 소득, 교육과 같은 인적 특성을 직접 지원함으로써 결과를 개선하고자 한다. 장소 기반 정책은 특정 지역의 인프라, 제도, 환경 조건을 개선함으로써 그 지역 거주자의 결과를 간접적으로 개선하고자 한다(Neumark & Simpson, 2015). 두 접근은 상호 배타적이지 않지만, 정책 자원의 배분과 개입 경로가 다르기 때문에 정책 집행의 관점에서 별도로 설계될 필요가 있다.

그러나 두 접근의 관계는 단순한 병렬이 아니다. 미국에 대한 Hindman(2000)의 시계열 분석은 소득·연령·교육이 지리보다 더 강한 예측 변수임을 보여주는 한편, 격차가 시간이 지남에 따라 고착화되는 패턴도 보고했다. 이는 개인 특성과 지역 구조가 시간에 걸쳐 상호작용할 수 있음을 함의한다. van Dijk(2020)는 동기 부족에서 물질적 접근, 기술, 그리고 실질적 이용으로 이행하는 순차 모형을 제안하며, 개인 요인과 구조적 요인이 누적적으로 작동함을 시사했다.

이러한 맥락에서 최근의 정책 및 학술 문헌은 사람 기반 접근과 장소 기반 접근의 양자택일에서 벗어나, 사람들의 경험을 장소의 맥락 안에서 이해하는 장소 민감 접근(place-sensitive approach)으로 이동해 왔다. 한 가지 예는 생활 경험과 장소 기반 통찰을 정량적 증거와 동등한 지식 형태로 인정하려는 시도이다(OECD, 2025). 이 접근은 사람 기반 요인과 장소 기반 요인이 분석적으로 분리될 것을 요구하는 동시에, 두 요인이 특정 장소에서 어떻게 결합되는지를 함께 고려할 것을 요구한다.

### 2.2.3 메타이론적 틀: 비판적 실재론의 계층화된 존재론

본 프로젝트는 분석 설계의 기본 원리를 채택하며, 그 설계의 메타이론적 토대를 비판적 실재론의 층화된 존재론(stratified ontology)에 둔다. Bhaskar(1975)에 따르면 세계는 세 영역으로 구성된다. 실재적(real) 영역은 관찰 여부와 무관하게 존재하는 구조, 메커니즘, 인과적 힘을 포함한다. 실제적(actual) 영역은 이러한 구조와 메커니즘이 특정 조건 아래에서 작동해 사건을 발생시키는 영역이다. 경험적(empirical) 영역은 발생하여 관찰 가능한 사건의 부분집합을 가리킨다. 실제적 영역의 모든 사건이 경험되는 것은 아니며, 관찰된 현상이 실재적 영역의 전체를 대표하지도 않는다. 디지털 불평등에 적용해 보면, Eynon(2023)은 이러한 비판적 실재론의 렌즈를 통해 인터넷 이용에서 관찰되는 차이가 항상 직접 가시화되지는 않는 기저의 구조와 제약에 의해 생성된다고 주장하며, 그 격차를 접근과 기술만의 문제를 넘어 재구성한다.

이러한 존재론적 구분은 역행추론(retroduction)의 논리를 수반한다(Bhaskar, 1975). 역행추론은 관찰된 경험적 사건으로부터 그러한 사건을 가능하게 한 구조와 메커니즘으로 거슬러 추론하는 방법이다. 이는 귀납이나 연역과 구별되며, "이 현상이 가능하기 위해서는 어떤 구조가 존재해야 하는가?"를 묻는다.

## 2.3 서울

### 2.3.1 도시로서의 서울

대한민국의 수도인 서울은 국토 면적의 0.6%에 전국 인구의 20%가 거주하는 고밀도 대도시이다. 높은 밀도와 기술 역량의 집적 덕분에 서울은 한국 디지털 전환의 최전선에 서 있다. 그러나 서울 내부를 면밀히 들여다보면 25개 자치구 각각의 인구 구조와 경제적 자족성과 결부된 의미 있는 격차가 드러난다. 본 절은 서울 자치구별 특성을 검토하고 서울의 인구 지리적 특징을 논의한다.

최근 서울의 인구 지리는 두 축, 즉 빠른 고령화와 1인 가구 확대를 따라 재편되어 왔다. 강북구(26.03%), 도봉구(25.85%)와 같은 북부 외곽 자치구는 이미 노년층 비중이 25%를 넘어 초고령사회 특성을 보이는 반면, 강남권과 주요 대학가 일대는 상대적으로 젊은 인구 구조를 유지해, 자치구 간 연령 분포의 불균형이 두드러진다. 이러한 인구학적 차이는 각 자치구의 평균 소득 수준 및 경제적 활력과 결합되어, 본 연구 3.1절에서 기술하는 Affordability 및 Devices 역량 차이와 연관된 맥락적 배경의 일부를 형성한다.

또한, 서울의 정책 집행 체계는 광역 단위의 서울특별시와 25개 기초자치구 사이의 유기적 협력을 통해 작동한다. 정책의 큰 방향은 서울시청이 정립한 하향식(top-down) 접근에 따르며, 시는 광역 단위 인프라 구축과 디지털 포용을 위한 표준 가이드라인 설정에서 중심적 역할을 한다. 동시에 서울은 시민참여예산제와 주민자치회 등 상향식(bottom-up) 의사결정 구조를 운영하며, 이를 통해 지역 주민이 일상에서 겪는 디지털 불편이 정책에 직접 반영된다. 자치구는 이러한 광역 계획에 기반해 지방의회 심의와 조례 제정을 거쳐 예산을 확정하고, 동주민센터를 통해 시민에게 밀착형 서비스를 최종 전달한다. 이런 의미에서 서울의 디지털 격차 해소 정책은 획일적인 중앙 집행이 아니라, 각 자치구의 행정 역량과 시민 참여를 결합한 다층적 과정을 통해 실현된다고 이해할 수 있다.

### 2.3.2 연결성 격차의 현황과 관련 법·정책

서울 디지털 연결성의 현재 상태는 자치구별 무선국 인프라 밀도의 격차에서 드러나듯이, 민간 주도 인프라 배치의 한계와 일치한다. 한국방송통신전파진흥협회의 2020-2025년 무선국 통계에 따르면, 서울의 통신 품질을 결정하는 핵심 인프라인 기지국과 이동 중계기는 수익성이 높고 트래픽이 집중되는 특정 지역에 압도적으로 편중되어 있다. 강남구(기지국 19,190개; 이동 중계기 2,711개), 송파구(기지국 14,751개)와 같은 동남권 자치구가 조밀한 네트워크를 보유한 반면, 도봉구(기지국 4,008개), 강북구(기지국 4,700개)와 같은 북부 외곽 자치구는 기지국 수에서 최대 4.8배의 격차를 보인다. 특히 소상공인 및 사물인터넷(IoT) 연결에 활용되는 간이 무선국의 경우, 강남구(24,236개)는 도봉구(1,224개)에 비해 약 20배 많아, 자치구 간 디지털 생태계에서 부익부빈익빈 양상이 이미 물리적 인프라 수준에서 고착되었음을 시사한다.

이러한 민간 인프라의 지역 불균형을 시정하고 시민의 기본적 통신권을 보장하기 위해, 서울시는 행정적 대안으로서 설비 기반 기간통신사업자로 직접 나서는 정책 결정을 내렸다. 2025년 개정된 전기통신사업법에 따라 서울시는 기간통신사업자 지위를 확보한 최초의 지방자치단체가 되었으며, 5,000km가 넘는 자체 광케이블 네트워크를 수익성을 이유로 민간 통신사가 충분히 투자하지 못한 소외 지역에 직접 연결함으로써 공공 와이파이와 IoT 인프라를 확장하고 있다. 서울시는 현재 전국 최대 규모인 34,000대 이상의 공공 와이파이를 운영하고 있으며, 이는 거리, 공원, 시장, 복지 시설, 시내버스 등에 폭넓게 설치되어 연간 약 2,000억 원의 시민 통신비 절감 효과를 창출하고 있다. 특히 약 7,000대의 시내버스에 설치된 모바일 와이파이는 시민이 일상에서 가장 직접적으로 체감하는 디지털 인프라가 되었다. 공공 와이파이 확대를 통해 서울시는 정보 접근 격차를 줄이고 매년 수천억 원에 이르는 가계 통신비 부담을 완화할 계획이다. 또한 민간 임대망에 대한 의존을 낮추고 확보된 자원을 시민 편익 사업에 재투자함으로써 지속 가능한 선순환을 구축하고자 한다.

이러한 인프라 격차 해소를 위한 정책 대응은 물리적 망 확충을 넘어, 시민의 실질적 비용 부담을 낮춤으로써 Affordability를 강화하는 방향으로 확장되어 왔다. 중앙정부와 지방정부는 이용자가 자신의 통신 소비 패턴을 이해하고 합리적 선택을 할 수 있도록 돕는 스마트초이스(Smart Choice) 플랫폼을 운영한다. 스마트초이스는 단말 보조금 확인, 맞춤형 요금제 추천, 미환급 환급금 조회 등 통합 서비스를 제공하여 정보 비대칭을 줄이고 가계 통신비 인하를 유도한다.

최근의 5G 요금제 개편은 고가 요금제 중심의 시장 구조를 중·저가 요금제 중심으로 전환시켰다는 점에서 특히 의미가 크다. 기존에 4만~5만 원대에 머물던 5G 최저 가격대는 3만 원대로 인하되었고, 실제 데이터 이용량에 대응하도록 30~100GB 구간의 중간 요금제가 새로이 보다 세분화되어 도입되었다. 이러한 개편은 이용자가 사용하지 않는 데이터에 대해 높은 비용을 지불해야 했던 구조적 모순을 해소했으며, 2024년에는 600만 명이 넘는 가입자가 중·저가 요금제로 전환하는 구체적 정책 성과로 이어졌다.

취약 계층 및 특정 연령대를 위한 맞춤형 전용 요금제도 크게 강화되었다. 청년층(만 34세 이하)을 대상으로 한국의 이동통신 3사는 같은 가격에 일반 요금제 대비 최대 2배의 데이터를 제공하며, 청년층의 높은 데이터 수요를 반영해 가입 가능 연령 상한을 만 29세에서 34세로 확대함으로써 대상 인구를 넓혔다. 또한 노년층(만 65세 이상)을 대상으로 LG U+는 노년층의 상대적으로 낮은 데이터 이용량과 높은 가격 민감도를 반영해, 일반 요금제보다 최대 20% 저렴한 연령대별(65세, 70세, 80세 등) 세분화 전용 요금제를 출시했다. 더불어 비대면 가입 추세에 맞추어, 온라인 전용 요금제는 이용자가 매장을 방문하지 않고 온라인 채널을 통해 가입할 경우 일반 요금제 대비 약 30% 저렴한 무약정 요금제를 도입함으로써 선택의 폭을 넓혔다.

그 결과, 서울은 스마트초이스를 통한 정보 투명성, 이동통신 3사의 합리적 요금 구조, 그리고 서울시의 공공 인프라 직접 공급이라는 3대 축의 틀을 통해 시민의 보편적 디지털 기본권을 지원하는 다면적 포용 정책을 구축하고 있다.

또한, 2026년 서울시 행정 서비스의 가장 큰 변화는 모바일 플랫폼의 단일화와 고도화이다. 서울시는 그동안 분산되어 있던 서울시민카드와 서울지갑 서비스를 2026년 3월 31일자로 종료하고, 그 기능을 통합 모바일 플랫폼인 서울ON에 통합한다. 도서관·체육시설 등 공공시설 이용에서부터 장애 여부나 국가유공자 자격 확인에 이르기까지 모든 것을 단일 앱에서 처리할 수 있게 함으로써, 이 변화는 서비스 분절이 만들어 낸 디지털 행정의 진입 장벽을 낮춘다. 특히 서울ON은 공공 마이데이터 연계를 통해 별도의 서류 제출 없이 취약 계층 자격 여부를 실시간으로 검증하는 기능을 강화함으로써 행정 효율성을 높인다.

나아가 서울시는 인프라 확충과 서비스 통합이 시민의 실질적 이용 역량 향상으로 이어지도록, AI 디지털 배움터를 중심으로 한 통합 포용 정책을 추진하고 있다. 서울의 10개 거점을 포함한 전국 단위의 디지털 배움터 거점을 활용해, 서울시는 노년층 및 디지털 취약 계층을 위한 상시 대면 교육을 제공하며, 기본적인 인터넷 이용 교육뿐 아니라 생활 밀착형 AI 체험 교육과 디지털 역기능 예방 교육도 제공한다. 특히 거동이 불편한 중증 장애인 등을 대상으로 한 "찾아가는 1:1 교육"을 확대하고, 온·오프라인을 연계한 수준별 맞춤형 교육 체계를 구축함으로써 교육 사각지대를 해소하고 있다. 이러한 접근은 하드웨어(인프라)와 소프트웨어(행정 서비스)가 갖추어져 있더라도 사람들이 이를 활용할 인적 역량을 갖추지 못하면 격차가 해소되지 않는다는 판단을 반영한다. 이와 같이 서울시는 법에 기반한 인프라 안정성 확보, 서울ON을 통한 서비스 전달 체계의 혁신, 그리고 교육을 통한 이용자 역량 향상이라는 3대 축의 틀을 통해 Universal Meaningful Connectivity(UMC)를 향해 나아가고 있다.

# 3. 분석

이 장의 세 분석은 순차적으로 연결된다. 먼저 본 연구는 서울의 자치구별 UMC 지수를 구축하여 상대적 격차를 측정한다 (3.1). 이어 이 결과를 자치구 수준 변수로 활용하여 개인 및 자치구 특성과 개인의 디지털 연결성 사이의 연관성을 분석한다 (3.2). 마지막으로 디지털 플랫폼 텍스트를 통해 정량 분석으로는 포착되지 않는 플랫폼 가시 사례를 식별한다. 이 순서는 측정, 설명적 분해, 그리고 탐색적 생활경험 신호 검출로 구성된다. 이는 자치구 수준 조건, 개인 수준 취약성, 일상적 서사를 서로 환원하지 않고 연결하기 위한 설계이다.

## 3.1 서울 UMC의 지역별 격차

### 3.1.1 측정 지표와 데이터

이 프로젝트에서 구축한 측정 지표는 UMC 6 개 차원의 개념적 정의에 기반한다. 6 개 차원은 Connectivity, Available for Use, Affordability, Devices, Digital Skills, Safety 로 분류되며, 각 측정 지표는 의미성과 보편성을 동시에 고려한 두 핵심 구성 요소에 따라 구체화되었다 (ITU, 2023). 본 분석에서 사용한 서울 UMC 6 개 차원의 측정 지표는 Table 1 에 제시되어 있다. 측정 지표 선정에서는 개념적 정의와의 정합성, 서울의 디지털 기술 환경, 데이터 가용성을 종합적으로 고려하였다. 분석에는 데이터 확보가 가능한 최근 두 시점, 즉 2023 년과 2024 년 자료를 사용하였다. 이 두 시점의 국가 행정 데이터와 관련 출처를 활용하였다. 다만 데이터 제공 시점의 제약으로 인해 Devices, Digital Skills, Safety 차원은 2023 년 조사 자료에 기반한 동일 값을 양 시점에 적용하였다.

Connectivity 지표는 Spectrum Resource Management System 의 이동통신 기지국 데이터, 한국 공공데이터포털의 통신 품질 데이터, 서울 자치구 경계, 주간 생활인구 분모를 사용하여 산출하였다. 서울처럼 고밀도 도시에서는 주민이 디지털 환경을 실제로 이용하는 장소가 등록 거주지와 다를 수 있다 (ref). 따라서 본 분석은 기지국 밀도를 자치구 수준의 인프라 조건으로 다루고, 중복 제거된 4G 와 5G 기지국 위치 및 인구 보정 밀도 측정치를 사용한다.

Available for Use 는 디지털 서비스의 실질적 이용 가능성을 측정하는 차원이다. 월별 모바일 데이터 사용량과 금융, 쇼핑, 동영상, 배달 등의 온라인 서비스 이용 일수는 SKT Telecom Data 에서 추출하였다. SKT 는 한국의 주요 이동통신 사업자인 SK Telecom 을 지칭한다. 동(洞) 단위 통신 지표는 주간 유동인구 가중치를 적용해 자치구 수준으로 집계하였고, 공공 접속 지점을 나타내기 위해 자치구가 보유한 Local Administrative Wi-Fi License Data 를 사용하였다.

Affordability 는 모든 주민이 통신 요금을 부담할 수 있는지를 직접 측정하는 지표라기보다, 양면적 맥락 지표로 구성되었다. 자원 측면에서는 Seoul Commercial Analysis Service 의 자치구별 월평균 가구 소득을 사용한다. 통신 가입 상품이 비교적 표준화되어 있고 많은 주민이 기본 요금제를 구매할 수 있다는 점에서, 자치구 평균 소득은 맥락적 자원 대리지표로만 해석해야 한다. 부담 측면은 서비스 접근성과 더 직접적으로 연결된다. 분석에서는 SKT Telecom Data 의 최근 통신 요금 연체율을 인구 가중으로 자치구 수준에 집계하고, 최종 값이 클수록 더 유리한 조건을 의미하도록 역코딩하였다.

Devices 는 디지털 기기의 보유와 사용 다양성을 측정한다. Seoul Citizens' Digital Competency Survey 를 기반으로 가구가 보유한 핵심 디지털 기기 종류 수와 개인이 실제로 사용하는 기기 종류 수의 자치구별 가중 평균을 산출하였다. ITU (2023) 에 따르면 보유와 사용은 휴대전화와 컴퓨터처럼 기기 특성에 따라 다르게 해석된다. 한국의 국내 맥락에서 컴퓨터 보유 여부는 디지털 연결성의 지표가 될 수 있다. 스마트폰 보급이 사실상 포화된 서울에서는 데스크톱, 노트북, 태블릿 등 보조 기기의 보유가 디지털 활용 역량의 차별 요인으로 작용한다 (Van Deursen & Helsper, 2015).

Digital Skills 는 Basic, Applied, Advanced, Cyber activity 의 네 개 하위 영역으로 구성된다. Basic 역량은 스마트 기기 조작의 기본 능력을, Applied 역량은 디지털 서비스 활용 능력을, Advanced 역량은 스마트 기기 활동에서의 숙련도를, Cyber activity 역량은 사이버 공간에서의 행위 능력을 측정한다. 각 하위 영역의 원점수는 가능한 최대 점수로 나누어 0-1 비율로 변환한 뒤, 자치구별 가중 평균을 산출하였다.

Safety 는 디지털 보안 행동과 보안 인식의 두 하위 지표로 구성된다. 보안 행동은 비밀번호 관리, 소프트웨어 업데이트, 2 단계 인증 사용 등 실제 보안 실천을 측정하며, 보안 인식은 개인정보 보호의 중요성에 대한 인식 수준을 측정한다. 각 하위 지표는 가능한 최대 점수 대비 비율로 변환한 뒤, 자치구별 가중 평균을 산출하였다.

### 3.1.2 측정 지표의 지수화

2.1 절의 이론적 논의에 따라, 서울은 다수의 UMC 기본 임계치가 이미 충족된 고연결 도시이므로 본 분석에서는 Min-Max 스케일링을 사용한다. 원래의 UMC 논리는 임계 기반이지만, 본 사례에서는 단순한 임계 검정만으로는 25 개 자치구 사이의 잔여 격차를 드러낼 수 없다. 따라서 이질적인 지표를 공통 척도에 놓은 뒤 각 자치구의 상대적 위치를 비교할 수 있도록, 서울 내부 Min-Max 스케일링을 통해 지표를 [0, 1] 구간으로 정규화한다. 정규화 값 x* 는 각 지표와 연도별로 다음과 같이 산출한다.

이렇게 얻어진 종합 점수는 6 개 차원 정규화 점수의 동일 가중 평균이다. 이는 각 지표가 서울 자체 범위에 맞추어 스케일링된 후 자치구의 서울 내 상대적 위치를 표현한다. 따라서 이 지수는 UMC 달성의 절대 수준이 아니라 서울 내부 상대 비교 지수로 해석해야 하며, 타 도시나 타 연도와 직접 비교 가능한 지표가 아니다.

### 3.1.3 자치구별 UMC 지수의 특성

KEY FINDING
측정된 서울의 UMC 격차는 다차원적이고 자치구별로 상이하다. 2024 년 종합 점수는 중랑구 0.279 에서 서초구 0.695 까지 분포하며, 잠정적 Digital Desert 군 (중랑구, 도봉구, 강북구, 구로구, 노원구) 은 단순한 물리적 연결성보다 Devices, Safety, Affordability 에서 가장 취약하다.

25 개 자치구의 2024 년 UMC 종합 지수는 0.279 (중랑구) 에서 0.695 (서초구) 까지 분포하여, 최상위와 최하위 자치구 사이에 약 2.5 배 격차가 존재하는 것으로 나타났다. 서울이 인터넷 보급률이 높은 도시라는 일반적 인식에도 불구하고, 하위 행정 단위에서 상당한 상대적 디지털 연결성 격차가 관찰되었다. 상위 5 개 자치구는 서초구 (0.695), 영등포구 (0.686), 마포구 (0.665), 서대문구 (0.615), 용산구 (0.603) 였다. 하위 5 개 자치구는 중랑구 (0.279), 도봉구 (0.303), 강북구 (0.380), 구로구 (0.389), 노원구 (0.389) 였다. 상위 자치구는 일반적으로 도심부 또는 강남 권역에 위치하여 밀집된 상업 활동, 높은 소득 수준, 풍부한 민간 디지털 인프라의 혜택을 받는 반면, 하위 자치구는 북부 외곽과 서남부에 집중되었다. 2024 년 종합 점수의 25 개 자치구 평균은 0.4905 였고, 6 개 차원 평균은 — Connectivity 0.512, Available for Use 0.490, Affordability 0.469, Devices 0.468, Digital Skills 0.500, Safety 0.505 — 서로 비교적 가까웠다. 이는 도시 전체 수준의 격차가 단일한 지배적 차원에 의해 결정된다기보다, 자치구별 강·약점이 차원에 걸쳐 어떻게 분포하는지에 따라 형성됨을 시사한다. 차원별 양극단은 이러한 이질성을 뒷받침한다. Connectivity 는 용산구 (0.884) 에서 정점에 달하고 노원구 (0.174) 에서 최저값을 보였다. Available for Use 는 광진구 (0.832) 가 선두였고 도봉구 (0.213) 가 최하위였다. Affordability 는 서초구 (0.993) 부터 강북구 (0.000) 까지 분포하였다. Devices 는 서초구 (0.988) 부터 구로구 (0.033) 까지 분포하였다. Digital Skills 는 관악구 (0.795) 가 최고치였고 중랑구 (0.242) 가 최저치였다. Safety 는 종로구 (0.984) 에서 최대치를 보였고 도봉구 (0.019) 가 최저점에 위치하였다.

종합 지수의 공간적 분포 (Figure 4) 를 살펴보면, UMC 지수 값이 높은 자치구는 두 영역에 집중되었다. 서초에서 영등포, 마포로 이어지는 한강·도심 회랑과, 용산, 종로, 중구를 중심으로 한 전통 도심 권역이다. 반면 북부 외곽 (강북구, 도봉구, 노원구) 과 서남부 (구로구, 금천구) 는 도시 평균 이하의 점수를 보였다. 이러한 공간적 분포는 서울의 사회경제적 자원 분포와 연관되었다. 즉, 디지털 격차가 도시 내의 구조적 공간 불평등과 밀접하게 연결되어 있을 가능성을 시사한다.

동시에 차원 수준 분석에서는 6 개 차원 모두에서 균일하게 높은 성과를 보인 자치구는 없었다 (Figure 3). 예를 들어 서울 북부의 은평구는 Connectivity 차원에서 0.781 의 점수로 상위에 위치했지만, Affordability 차원에서는 0.328 로 하위권에 머물렀다. 마찬가지로 관악구는 Digital Skills 차원에서 0.795 로 1 위였으나, Affordability 에서는 0.152 로 최하위 군에 속하였다. 이러한 다차원적인 자치구 간 불평등은 정책적으로 의미 있는 차원별 취약성이 종합 순위만으로는 포착될 수 없음을 보여 준다.

## 3.2 개인 및 자치구 특성과 디지털 연결성

본 연구의 데이터는 개인 (Level 1) 이 자치구 (Level 2) 내에 묶여 있는 위계 구조를 가진다. 자치구-연도 UMC 점수는 응답자의 거주 자치구 및 조사 연도를 기준으로 결합되며, 서울 25 개 자치구에 대해 임의 절편이 추정된다. 이런 구조에서는 통상적 OLS 회귀가 관측치 독립 가정을 위배하고 자치구 수준 예측변수의 표준오차를 과소추정하여 제1종 오류를 부풀릴 수 있다 (Raudenbush & Bryk, 2002). 따라서 본 연구는 분산을 자치구 내 및 자치구 간 구성요소로 분해하고 개인 수준과 자치구 수준 변수를 동시에 투입할 수 있는 2 수준 위계선형모형 (HLM) 을 채택하였다.

두 번째 분석에 사용된 Seoul Survey 는 서울특별시가 2003 년부터 매년 20,000 가구를 대상으로 수집해 온 행정 데이터이다. 자료 수집은 매년 8-9 월에 비대면과 대면 조사를 결합하여 수행된다. 공식 Seoul Survey 표본설계 문서와 통계청 승인 고시에 따르면, 본 분석에 사용된 개인 수준 파일은 시민조사 응답자 구성요소에 해당한다. 매년 만 15 세 이상 서울 시민 5,000 명이 조사되며, 따라서 2023-2024 년 통합 파일은 10,000 명의 응답자를 포함한다. 시민조사는 Seoul Citizen Panel 을 표집 틀로 사용한다. 이 패널은 원래 서울 주민등록 기록과 건축물대장 기록 사이에 연결된 가구 주소로 구축되었으며, 자치구, 행정동, 주택 유형으로 층화되어 있다. 연간 조사에서는 활성 패널 내에서 자치구별 주택유형 층 내 무작위로 가구를 선정하며, 선정된 가구에 만 15 세 이상 거주자가 여러 명일 경우 가장 최근 생일 규칙에 따라 한 명의 응답자를 선정한다.

따라서 HLM 의 65 세 이상 범주는 응답자의 관측된 연령에서 도출된 분석적 임계값이지, 시민조사 구성요소 안에 65 세 이상이 별도로 과대표집되었다는 증거가 아니다. 2023 년 가구 표집 문서는 선정된 군집 내 가구를 가구원 수와 65 세 이상 거주자 비율로 정렬한 뒤 계통적 가구 선정을 수행했음을 밝히고 있으며, Seoul Survey 가중치는 자치구, 성별, 광역 연령군에 따라 개인 추정치를 보정한다. 이러한 절차는 표본을 모집단 구조에 맞추는 데 기여하지만, 본 분석에서 노년층을 독립적인 표집 층으로 만들지는 않는다.

### 3.2.1 변수 구축

먼저, 분석의 종속변수인 디지털 이용 점수는 Seoul Survey 응답 항목 중 디지털 서비스 이용 빈도에 관한 8 개 문항의 합산 점수이다. 8 개 문항은 생활정보 서비스, 전자상거래 서비스, 금융거래 서비스, 공공 서비스, 커뮤니케이션 서비스, 여가 서비스, 교육·학습 서비스, 모바일 결제로 구성된다. 이는 통화나 문자 같은 기본적 연결 행위가 아니라 디지털 기기를 매개로 한 실질적인 추가 이용에 해당하므로, 서울의 고도화된 디지털 환경에 부합하는 지표로 판단되었다. 각 문항은 원래의 4 점 서열 척도에서 0-10 연속 척도로 변환되어, 0 점에서 80 점 범위의 연속 변수가 산출되었다. 점수가 높을수록 개인이 보다 다양한 디지털 서비스를 이용하고 원하는 서비스를 자유롭게 사용할 수 있음을 의미한다.

본 분석의 독립변수는 개인 특성 변수와 자치구 특성 변수로 구분된다 (Table 2). 개인 특성 변수는 성별, 연령, 직업, 학력, 소득 수준, 장애 여부로 구성되며, 이는 모두 개인의 디지털 이용과 관련된 이론적으로 의미 있는 상관 변수이다. 학력은 중졸 이하, 고졸, 대졸 (참조 범주), 대학원 졸 이상의 네 범주로 구분하였다. 직업은 관리직·전문직, 사무직 (참조 범주), 생산·기능직, 기타 (학생, 주부, 무직) 의 네 범주로 구분하였다. 주택 유형은 아파트 (참조 범주), 단독주택, 다세대·다가구 주택, 기타의 네 범주로 분류하였다. 연령과 소득은 절편의 해석 가능성을 높이기 위해 전체평균 중심화하였다. 추가로 1 인 가구 더미, 장애 여부 더미, 연도 더미 (참조: 2023) 를 포함하였다.

Level 2 변수로는 지역 디지털 맥락 조건을 기술하는 3.1 절에서 구축된 UMC 차원 점수 3 개, 즉 Connectivity, Available for Use, Affordability 를 사용하였다. 이러한 선택은 Devices, Digital Skills, Safety 가 Seoul Citizens' Digital Competency Survey 의 개인 응답으로부터 자치구 수준으로 집계된다는 점을 반영한다. 이로 인해 발생하는 내생성 문제를 피하기 위해 이 세 차원은 주 분석에서 제외하였다. 모든 Level 2 UMC 변수는 Min-Max 정규화로 [0, 1] 범위로 표준화되었고 자치구 및 조사 연도 기준으로 결합되었다.

분석은 다섯 단계의 모형으로 진행된다. 무조건 모형을 추정하고 다층 분석의 사용 근거로서 급내상관계수 (ICC) 를 산출한 뒤, 개인 수준 변수를 포함한 M1, 세 개의 자치구 수준 UMC 차원을 추가한 M2, 그리고 선택된 상호작용항을 포함한 M3 를 순차적으로 추정한다. 유지된 상호작용항은 2.1 절과 2.2 절의 이론적 틀을 취약성 메커니즘으로 옮긴다. 낮은 학력은 접근을 의미 있는 이용으로 전환하는 데 대한 제약을 나타내고, 연령은 누적된 역량 한계를 포착하며, 자치구 연결성은 학력 자원이 낮은 주민에게 다르게 작용할 수 있는 맥락 조건으로 검증된다 (Hargittai, 2010; Helsper, 2012; van Deursen & van Dijk, 2015; Mossberger et al., 2006; van Dijk, 2020). 주 완전 모형 (M3) 은 65 세 이상 더미와 노년 특화 상호작용을 모두 제거하고, 연속 연령 항과 연령 × 저학력 상호작용을 통해 노년 패턴을 포착한다. Model 4 는 보조적 연령 임계 설정으로 추가된다. 연속 연령을 제거하고 65 세 이상 더미를 포함하며, 65 세 이상 × 저학력 상호작용을 추가한다.

### 3.2.2 분석 결과

KEY FINDING 

HLM 결과에서 개인 취약성이 지배적 구조를 이룬다. 자치구 수준 맥락 연관성은 주 모형에서 매우 작은 반면, 낮은 학력, 연속적 경사로서의 고령, 1 인 가구, 단독주택 또는 다세대 주택, 그리고 연령 × 저학력 패턴이 디지털 이용에서 관찰된 주요 차이를 설명한다. Model 4 는 65 세 이상 임계값이 낮은 디지털 이용과 연관되어 있다는 해석과 일관된 결과를 보이며, 낮은 학력과 결합될 때 특히 그러하다. 다만 연속 연령 설정보다 적합도가 떨어지므로 보조적 민감도 모형으로 다룬다. AIC 는 Model 4 가 81,661 인 반면 Model 3 는 81,315 이다.

분석에 사용된 표본은 2023-2024 년 Seoul Survey 응답자 총 10,000 명으로, 25 개 자치구 각각에서 평균 400 명이 표집되었다. Table 3 은 주요 변수의 기술통계량을 제시한다. 개인 디지털 연결성의 평균값은 80 점 만점에 60.88 로, 서울 주민의 디지털 연결성 수준이 높음을 보여 준다. 종속변수가 음의 왜도를 보일 가능성이 있으나, 제도적 표준 척도와 10,000 에 가까운 표본 크기를 고려하여 척도 선형화된 원점수를 사용해 분석하였다.

순차적 추정 결과는 디지털 이용의 자치구 간 변동이 작음을 보여 준다. 무조건 모형의 ICC 는 0.49% 로, 전체 분산 중 자치구 사이에 놓인 비중이 매우 작음을 의미한다. 더 중요한 실질적 함의는 수치 자체가 아니라 그 함의에 있다. 즉, 관찰된 격차의 대부분은 학력, 연령, 가구, 소득, 직업, 주택 특성이 서로 다른 주민들 사이에서 자치구 내부에 구조화되어 있다는 점이다. 설계 효과는 여전히 다층 모형 사용을 뒷받침하지만, Level 2 계수는 제한된 맥락 연관성으로 읽어야 한다.

개인 특성 중 학력과의 연관성이 가장 두드러진다. 중졸 이하 응답자는 대졸자에 비해 예측 디지털 이용이 훨씬 낮으며, Model 3 의 연령 × 저학력 상호작용은 최저학력군에서 연령 경사가 더 가파름을 보여 준다. Model 4 는 이 패턴을 정책 가독성이 높은 임계 형태로 재구성한다. 65 세 이상 응답자는 점수가 더 낮으며, 65 세 이상 × 저학력 상호작용은 추가적 불리함을 시사한다. Model 4 의 적합도가 Model 3 보다 약하므로, 이 임계 결과는 연속 연령 모형을 대체한다기보다 노년층 취약성에 대한 보조 해석으로 읽어야 한다.

Table 4 는 또한 HLM 결과에서 가구 구성과 통제변수가 가시적으로 유지되어야 하는 이유를 보여 준다. 1 인 가구는 Model 1-3 에서 약 1.7-1.8 점 낮게 나타나, 주 연속 연령 설정에서 의미 있는 연관성을 보인다. 그러나 Model 4 에서는 65 세 이상 임계값과 그 저학력과의 상호작용이 투입된 뒤 1 인 가구 계수가 작아지고 통계적으로 유의하지 않게 된다. 이는 가구 구성 패턴의 일부가 완전히 별개의 임계 효과로 작동하기보다는 노년층 취약성과 중첩된다는 점을 시사한다. 통제변수 중 여성은 남성보다 약 2.6-2.8 점 높게 나타나고, 소득 계수는 양수이지만 실질적으로 작으며, 관리직·전문직은 주 설정에서 화이트칼라 노동자보다 높은 점수를 보인다. 단독주택과 다세대 주택 거주자는 아파트 거주자보다 낮은 점수를 보이며, 연도 지표는 결과와 실질적으로 연관되지 않는다. Model 1-3 의 양의 장애 계수는 일반적인 장애의 이점에 대한 증거가 아니라, 표본 특이적 연관성으로 신중히 해석해야 한다.

자치구 수준에서 UMC 차원은 강하거나 일관된 평균 효과를 보이지 않는다. 주 3 차원 설정에서 Connectivity 와 Affordability 는 Model 3 에서 견고하게 유의하기보다 한계적이며, 추정은 자치구 간 분산이 매우 작고 Level 2 검정력이 제한된 환경에서 산출된다. Model 4 에서는 Affordability 가 통계적으로 유의해지지만, 이는 전반적 적합도가 더 약한 보조 임계 설정에서 발생한다. 더 안전한 해석은 자치구 지표가 자치구 조건 자체가 개인 결과를 야기한다는 인과적 증거를 제공하기보다는, 취약한 주민 주변의 맥락 조건을 기술한다는 것이다.

교차 수준 상호작용 결과는 신중히 읽어야 한다. Model 3 에서 유지된 자치구 수준 조절항은 Connectivity × 저학력이다. 그 계수는 양수이지만 전체 표본에서는 통계적으로 유의하지 않으며, 저학력 자치구 제외 민감도 검정에서는 양수를 유지하면서 유의해진다. Model 4 에서는 동일한 Connectivity × 저학력 항이 한계적으로 되는 반면, 65 세 이상 × 저학력 상호작용은 음수이며 통계적으로 유의하다. 이 패턴은 취약성에 초점을 둔 해석을 뒷받침하지만, 자치구 수준 조절의 증거는 개인 취약성의 증거보다 여전히 약하고 덜 안정적이다.

개인 수준에서 유지된 상호작용 결과는 이중 취약성 패턴을 시사한다. Model 3 는 낮은 학력과 디지털 이용 사이의 음의 연관성이 연령과 함께 더 커진다는 점을 보여 주며, Model 4 는 낮은 학력과 65 세 이상이 결합하여 추가적 불리함을 만들어 낸다는 임계 형태의 유사한 해석과 일관된 결과를 보인다. 이 발견들은 연령과 학력의 제약이 서로 가중되는 주민에게 초점을 둔 정책적 해석을 뒷받침하면서도, HLM 이 일반적인 자치구 수준 인과 효과를 식별한다는 주장은 피한다.

## 3.3 다중 에이전트 질적 연구

3.1절과 3.2절은 행정 데이터와 서베이 데이터를 활용해 서울의 디지털 격차를 양적으로 측정하고, 개인 특성과 자치구 수준 맥락 조건의 역할을 분리해 살펴보았다. 그러나 양적 지표만으로는 주민이 일상 속 어느 시점에, 어떤 맥락에서 디지털 연결성의 결핍을 경험하는지 포착하기 어렵다. 따라서 본 절에서는 플랫폼 텍스트를 활용한 생활 경험 신호 탐지, 즉 이용자의 자발적 게시물을 통해 가시화되는 경험을 식별하는 작업을 수행한다. 본 절의 목적은 양적 분석 결과를 맥락화하고 예시적으로 보완하는 것이며, 확증적 증거나 모집단 수준의 유병률 추정을 제시하는 것이 아니다.

### 3.3.1 당근 플랫폼의 특성

비정형 텍스트의 출처는 한국의 지역 커뮤니티 플랫폼인 당근(Danggeun)의 동네생활 게시물이다. 당근을 선택한 이유는 세 가지다. 첫째, 각 게시물에는 행정동 및 자치구 수준의 지리 메타데이터가 자동으로 부여되어, 3.1절에서 구축한 UMC 지수와 연계가 가능하다. 둘째, 플랫폼 이용자가 서울 25개 자치구 전반에 분포해 있어 자치구 수준 집계가 기술적으로 가능하다. 셋째, 게시물이 일상 속에서 마주친 문제를 자발적으로 기록하기 때문에, 서베이가 놓칠 수 있는 생활 경험 신호를 가시화할 수 있다. 이러한 장점이 당근 게시물을 서울 주민 전체에 대해 대표성을 갖게 하는 것은 아니며, 본 분석은 게시 행위를 선택한 플랫폼 이용자의 경험만을 관찰한다.

### 3.3.2 분석 구조

본 절의 분석은 당근 게시물을 단순히 수집된 텍스트로 다루지 않고, 디지털 연결성 결핍을 포착할 수 있는 잠재적 생활 신호로 해석한다. 그러나 게시물 하나가 곧바로 지역 수준의 정책 증거가 되는 것은 아니다. 게시물은 특정 플랫폼 이용자가 특정 상황에서 자발적으로 남긴 기록이기 때문에, 본 연구는 개별 게시물을 대표성 있는 관측값으로 간주하지 않는다. 대신 게시물이 어떤 UMC 차원과 관련되는지 먼저 분류하고, 이후 그 신호가 자치구·차원 수준에서 행정자료 기반 기대값과 어떻게 다른지 탐색적으로 집계한다. 동시에 일부 게시물은 사례 수준에서 어떤 결핍이나 경로 단절을 드러내는지 해석된다.

이러한 절차에서 LLM을 사용하는 이유는 게시물의 의미를 모델이 자유롭게 결정하도록 하기 위해서가 아니다. 당근 게시물은 구어체 한국어, 은어, 지역적 지시 표현, 일상적 불만, 서비스 이용 경험이 뒤섞인 짧은 텍스트로 구성된다. 이때 디지털 연결성은 글의 중심 주제로 명시되기보다 생활 문제의 일부로 삽입되는 경우가 많다. 단순 키워드 검색은 후보 게시물을 넓게 찾아낼 수 있지만, 해당 글의 핵심 문제가 디지털 연결성인지 안정적으로 판정하기 어렵다. 토픽 모델은 반복되는 주제를 탐색하는 데 유용하지만, 발견된 주제가 사전에 정의된 UMC 차원 중 어디에 해당하는지 스스로 판단하지는 못한다. 따라서 본 연구에서 LLM은 사전에 정의된 prompt, seed list, 코드북에 따라 게시물을 검토하는 구조화된 코딩 보조자로 사용된다. 여기서 prompt는 LLM에 제공되는 고정 지시문, seed list는 후보 게시물 수집에 사용된 초기 단어·표현 목록, 코드북은 분류 기준표를 의미한다.

본 절의 하류 해석 구간은 하나의 순차적 단계가 아니라, 동일한 분류 결과에서 출발하는 두 개의 병렬 구성요소로 이루어진다. 첫 번째 층은 필터링된 후보 게시물을 대상으로 UMC 관련성 여부와 6개 UMC 차원을 분류한다. 두 번째 층은 관련 게시물을 두 방향으로 해석한다. 하나는 탐색적 EB 수축률 모형을 통해 자치구·차원 수준의 플랫폼 가시 신호를 집계하는 방향이고, 다른 하나는 세 개의 제한된 가설 agent와 판단 종합기를 통해 게시물 수준 사례를 해석하는 방향이다. agent는 여기서 독립적인 행위자가 아니라, 동일한 게시물을 서로 다른 질문으로 읽는 가설별 해석기를 의미한다. Figure 7은 원시 플랫폼 기록에서 결정론적 필터링, LLM 보조 관련성 코딩, 자치구·차원 EB 수축률 집계, 게시물 수준 추론 해석에 이르는 전체 파이프라인을 요약한다.

분류 단계의 목적은 게시물이 후보 신호 기반에 포함될 수 있는지를 판단하는 것이다. 각 게시물은 먼저 UMC 관련성 여부를 기준으로 검토되며, 관련 게시물은 연결 품질, 가용성, 비용 부담, 기기 접근, 디지털 역량, 안전·보안의 6개 차원 중 하나 이상으로 분류된다. 이 범주는 개별 이용자의 속성을 규정하기 위한 것이 아니라, 게시물 속에 나타난 디지털 연결성 문제의 유형을 기술하기 위한 것이다. 예컨대 높은 통신요금에 관한 게시물은 Affordability 사례가 될 수 있지만, 비싼 식당에 관한 불만은 그렇지 않다. 피싱 링크에 관한 게시물은 Safety 사례가 될 수 있지만, 모든 중고 거래 분쟁이 곧 사이버보안 문제는 아니다. 이러한 예시는 few-shot prompt, 즉 예시 기반 고정 지시문을 통해 범주의 실질적 경계를 안정화하는 데 사용된다.

게시물은 LLM이 라벨을 산출했다는 이유만으로 증거로 채택되지 않는다. 전처리 단계에서는 사람이 작성한 seed list와, 새로운 키워드가 거의 추가되지 않을 때까지 확장한 키워드 사전을 이용해 후보 자료를 좁힌다. 분류 단계에서는 모든 게시물에 동일한 관련성 질문을 적용한다. 핵심 쟁점이 디지털 문제인지, 디지털 관련 단어를 제거해도 글이 여전히 같은 의미로 성립하는지, 그리고 문제 해결이 인프라, 기기, 역량, 비용 부담, 가용성, 보안 측면의 변화를 요구하는지를 검토한다. 해석 단계에서는 추가 요건이 부여된다. 모든 주장은 짧은 텍스트 증거 조각에 연결되어야 하며, 편의, 일상적 사회 상호작용, 일반적인 서비스 불만족과 같은 대안적 설명을 먼저 검토한 뒤에야 해당 게시물을 UMC 사례로 다룬다. 이 설계에서 prompt는 자유로운 대화가 아니라 코드북, 즉 분류 기준표로 기능하며, 동일한 점검을 동일한 순서로 요구하고, 근거가 부족한 추론을 제한하며, 어떤 게시물이 왜 유지·제외되었는지를 추적할 수 있는 audit trail(판단 추적 기록)을 남긴다.

EB 수축률 모형은 이러한 분류 결과를 자치구·차원 수준에서 집계하기 위해 사용된다. 베이지안 모형을 사용하는 이유는 지역밀착형 게시글에서 특정 UMC 차원으로 분류된 신호가 행정자료 기반 기대값에 비해 특정 지역에서 상대적으로 강하게 나타나는지를 탐색적으로 확인하기 위함이다. 개별 게시물은 여러 차원에 동시에 포함될 수 있고, 게시글 수 자체도 지역별로 균등하지 않다. 따라서 단일 게시물을 곧바로 지역 수준의 정책 근거로 해석하기에는 한계가 있으며, 본 연구는 이를 보완하기 위해 행정자료 기반 사전 기대와 플랫폼에서 관찰된 게시 신호를 결합하는 방식으로 집계한다.

모형에서 사전율은 Part 1에서 구축한 자치구·차원 결핍 지표가 예측한 기대 민원율이다. 단위는 인구 10만 명·년 기준이다. 사후율은 작성 시점의 시간대 생활인구 노출값인 living_pop_exposure_gu_hourly로 보정한 실제 게시글 율이다. posterior shift는 사후율에서 사전율을 뺀 값으로 정의되며, 플랫폼에서 관찰된 신호가 행정자료 기반 예측에서 벗어나는 방향을 기록한다. 양의 값은 해당 차원의 주민 게시 신호가 행정자료 기반 기대보다 강하게 나타난다는 뜻이고, 음의 값은 약하게 나타난다는 뜻이다. 다만 차원마다 절대율의 크기가 다르기 때문에, 차원 간 비교에는 차원 내 표준화 값인 z_shift를 사용한다. 본 연구는 |z_shift| > 1.5인 자치구·차원 조합을 두드러진 괴리 셀로 읽는다. 이 모형은 자치구 맥락이 개별 경험을 야기했음을 보여 주지 않으며, 그 역할은 질적 해석과 3.2절의 양적 신중함을 함께 읽어야 할 지점을 식별하는 데 있다.

EB 수축률 모형과 병렬로 배치되는 agent 기반 해석층은 EB 모형의 출력값을 다시 설명하는 절차가 아니라, 게시물 수준의 해석적 표본을 읽는 절차이다. EB 모형이 자치구·차원 수준에서 플랫폼 가시 신호의 방향과 크기를 요약한다면, agent 기반 해석은 개별 게시물이 어떤 종류의 결핍, 경로 실패, 또는 제도적 차단을 시사하는지 설명한다. 따라서 두 층은 같은 게시물 기반에서 출발하지만 관찰 단위가 다르다. EB 수축률 집계의 정책 해석 단위는 자치구·차원 조합이고, agent 기반 해석의 관찰 단위는 개별 게시물이다. Table 5는 이 두 분석 단위를 구분하여 제시한다.

few-shot 분류와 reasoning-guided 해석의 차이도 이 지점에서 중요하다. few-shot 분류는 다수의 게시물을 사전 정의된 결정 범주와 6개 UMC 차원에 안정적으로 배치하기 위한 절차이다. 이 단계에서는 유사하지만 서로 다른 사례의 경계를 보여 주는 예시가 유용하다. 반면 reasoning-guided 해석은 이미 관련 있는 것으로 판정된 게시물이 왜 디지털 결핍 사례로 읽힐 수 있는지를 설명하도록 요구한다. 동일한 UMC 차원에 속한 게시물이라도 서로 다른 기제를 함의할 수 있기 때문이다. 따라서 reasoning prompt는 게시물에서 해석에 이르는 경로를 명시하도록 설계된다. 무엇이 의외인지, 어떤 기저 조건이 그 상황을 설명할 수 있는지, 정상적인 접근 경로라면 어떤 일이 일어났어야 하는지, 이용자의 경로가 어디에서 무너졌는지, 그리고 어떤 경쟁적 설명이 더 설득력 있을 수 있는지를 검토한다.

reasoning-guided로 작성된 세 개의 추론 agent는 동일한 게시물을 서로 다른 제약 하에서 읽는 세 가지 렌즈로 이해하는 것이 적절하다. abductive 렌즈는 관찰된 상황에서 출발해 그것을 설명할 수 있는 기저 조건을 묻는다. forward 렌즈는 특정 조건이 존재한다면 텍스트에 어떤 신호가 나타나야 하는지를 묻는다. sequential 렌즈는 이용자의 접근·이용·해결 경로를 재구성하고, 그 경로가 어느 지점에서 실패했는지를 묻는다. 이는 동일한 모델이 단순히 세 번 답을 생성한 것이 아니라, 동일한 텍스트 증거를 서로 다른 질문 구조 아래에서 읽는 절차이다. 서로 다른 렌즈가 같은 해석을 가리킬 때 최종 판단은 상대적으로 강해지고, 서로 어긋날 때는 더 신중한 판단이 요구된다.

맥락 정보는 단계적 입출력 구조를 통해 관리된다. 먼저 각 가설 agent는 게시물 텍스트, 기본 메타데이터, 일반적 코딩 체계만 제공받는다. 이 단계에서는 자치구 프로필, 생활인구 맥락, 결핍 유형, 활성 범주 정의, 그리고 다른 agent의 출력이 의도적으로 차단된다. 이후 종합 단계에서 판단 종합기는 세 가설 출력과 함께 자치구 맥락, 결핍 유형, 활성 범주 집합을 받아 수렴, 불일치, 비구조적 대안, 최종 코드를 평가한다. 확률 공간의 관점에서 이는 각 게시물에 대해 완전히 추정된 수치적 posterior를 갖는다는 주장이 아니라, 단계적 조건화 설계로 이해할 수 있다. 가설 agent는 제한된 정보 집합 I_R 하에서 출력을 생성하고, 판단 종합기는 확장된 정보 집합 I_J 하에서 분류를 평가한다. 이러한 분리는 맥락적 prior가 텍스트의 첫 읽기에 누출되는 것을 줄이고, 이후 맥락 사용을 명시적이고 감사 가능하게 만든다. ### 3.3.3 데이터 수집 및 전처리 과정

원시 데이터는 2024년 5월부터 2026년 3월까지 서울 25개 자치구를 대상으로 작성된 동네생활 게시물을 크롤링한 것이다. 본 분석의 전처리는 세 단계로 진행되며, 각 단계의 표본 규모는 분석 산출 파일에서 직접 확인되는 값을 보고한다. 첫째, 크롤링한 전체 게시물에서 본문 접근이 불가능한 게시물(계정 탈퇴·차단으로 본문이 비어 있는 경우), 중복 게시물, 광고·홍보성 게시물을 제외한다. 둘째, 남은 게시물에 UMC 6개 차원의 seed list와 확장된 키워드 사전을 적용해 후보 게시물 149,733건을 추출한다. 셋째, 1단계 LLM 판정이 이 후보를 한정된 관련성 범주(관련 Y, 불확정 ?, 무관 N)로 분류하며, 행정동 매칭이 가능한 131,792건이 분류 완료 표본을 구성한다. 분류 완료 표본의 관련성 분포는 관련(Y) 7,136건, 무관(N) 122,509건, 불확정(?) 2,147건이다. 차원 부여는 관련(Y)으로 분류된 게시물에 한해 수행되며, 한 게시물에 둘 이상의 차원이 부여될 수 있다.

여기서 한정되었다는 것은 코딩 절차가 사전 정의된 관련성 및 차원 코드북에서만 선택할 수 있고 라벨을 자유롭게 만들어낼 수 없음을 뜻한다. 한정된 범주는 개별 이용자가 아닌 디지털 연결성 신호의 유형을 기술한다. Table 6은 이 세 단계의 표본 구성을 요약한다.

당근 데이터는 구조적 한계를 가진다. 플랫폼 이용자 자체가 해당 서비스를 사용할 수 있을 만큼의 디지털 접근성을 갖춘 집단으로 제한되기 때문에, 디지털적으로 배제된 집단의 경험은 구조적으로 과소대표된다. 이는 선택 편향 위험을 발생시킨다. 기기, 연결성, 역량, 비용 부담 측면에서 심각한 장벽을 가진 주민은 애초에 온라인에 게시할 가능성이 낮을 수 있다. 따라서 플랫폼 가시 신호는 서울의 모든 디지털 결핍에 대한 대표 표본이 아니라 하한이며 경험 선택적인 증거로 다뤄야 한다. 게시물의 시간적·공간적 분포 역시 균일하지 않으므로, 텍스트 기반 결과는 모집단 수준의 유병률 추정이 아니라 플랫폼 가시 생활 경험 신호로 해석되어야 한다.

지역·차원 집계 작업 흐름에서, 동 수준 연계는 자치구 집계 이전의 전처리 단계로 처리되었다. 분류된 131,792행 모두를 행정동 후보와 매칭하였고, 모호한 법정동 매핑은 단일 동으로 강제 할당하지 않고 동일 할당 가중치로 보존하였다. 이로써 매핑 정밀도를 실질적 발견으로 전환하지 않으면서 집계 총량을 보존한다. 생활인구 연계는 집계 기반의 일관성을 유지하기 위해 정확한 날짜·시간·행정동 수준에서 점검되었다. 그 결과 밀도 입력은 420개 행정동, 580개 날짜에 걸쳐 약 14.6만 개의 정확 매칭 행을 포함한다. 일부 누락된 공식 서울 생활인구 셀은 보간 없이 제외하였다. 이러한 점검은 이후 EB 수축률 집계 구성을 위한 집계 기반을 확립하는 품질 통제이며, 핵심 결과가 아니다.

### 3.3.4 분석 결과

핵심 발견
EB 수축률 모형은 행정 지표와 플랫폼 목소리 사이의 어긋남이 Devices·Digital Skills 차원에서 가장 크고, Connection Quality·Available for Use·Affordability는 사전 근처에 머문다는 것을 보여 준다. 가장 두드러진 숨은 호소(점수는 양호하나 목소리가 강한 셀)는 서초구 Devices(z = +2.46, shift +16.3 / 100k person-years)이며, 가장 두드러진 조용한 결핍(점수는 나쁘나 목소리가 약한 셀)은 구로구 Devices(z = −2.06, shift −56.8 / 100k)이다. 이는 탐색적 어긋남 신호이지 유병률 추정이나 행정 지표 오류 증명이 아니다.

Figure 10은 EB 수축률 모형의 실질적 결과를 보고한다. 가로 막대 그림은 차원 내 표준화 z_shift 기준으로 가장 두드러진 18개 자치구·차원 셀을 정렬하며, 오른쪽(파랑)은 목소리가 예측보다 강한 셀, 왼쪽(빨강)은 약한 셀이다. 중요한 해석 단서: 이 그림에 오른 셀은 결핍이 가장 심한 곳이 아니라 지표와 목소리의 거리가 가장 먼 곳이다. 막대 길이는 결핍 크기가 아니라 예측과 실제의 거리를 나타내므로, 이 그림은 추가로 들여다볼 후보 셀의 목록으로 읽어야 한다.

차원 수준에서 여섯 차원은 두 무리로 나뉜다. 첫 번째 무리인 연결품질(CQ)·이용가능성(AFU)·경제성(AFF)에서는 사후가 사전 가까이 머문다. 자치구 간 게시글 율의 실제 차이가 작아 모형이 사전을 강하게 신뢰하며, shift 폭도 좁다. 인프라나 요금처럼 지역 단위로 측정되는 결핍은 Part 1 지표가 주민 체감을 비교적 잘 대리함을 시사한다. 두 번째 무리인 기기(Devices)와 디지털 역량(Digital Skills)에서는 shift 폭이 매우 넓어 어떤 셀은 −50 / 100k 아래로, 다른 셀은 +15 / 100k 이상으로 이동한다. 사전과 사후가 거의 따로 움직이는 신호다. 기기 보유와 역량은 가구·개인 수준에서 결정되는 속성이라 지역 평균 지표가 체감을 잘 예측하지 못하는 차원이며, 행정 지표가 놓치는 영역을 플랫폼 목소리가 비추는 지점이 바로 여기다.

숨은 호소(점수는 양호하나 목소리가 강한 셀) 중 가장 두드러진 셀은 서초구 Devices(z = +2.46, +16.3 / 100k)다. 서초구는 소득·인프라 지표가 높아 기기 결핍이 사실상 없다고 나오지만, 실제 기기 관련 게시글 율이 가장 강한 양의 어긋남을 만든다. 지역 평균이 좋아도 그 안의 호소가 사라지지 않음을 보여 준다. 관악구는 이용가능성(AFU, z = +2.15, +4.6 / 100k)과 연결품질(CQ, z = +1.83, +5.3 / 100k) 두 차원에서 동시에 예측을 웃도는 목소리를 낸다. 1인가구·청년·대학가가 밀집한 지역 특성과 맞물려 지역 평균 지표가 덜 포착하는 체감 결핍이 여러 차원에 걸쳐 있을 가능성을 시사한다. 송파구 Digital Skills(z = +1.70, +15.4 / 100k)와 Safety(z = +1.66, +12.0 / 100k)는 게시글 수가 수백 건으로 커서 통계적으로 가장 단단한 신호에 든다. 이러한 사례는 플랫폼 가시 도움 요청이나 불편 신호가 측정된 prior가 시사하는 것보다 강한 지점을 나타낸다.

조용한 결핍(점수는 나쁘나 목소리가 약한 셀)은 해석에 더 조심스럽다. 구로구 Devices(z = −2.06, −56.8 / 100k)와 강동구 Devices(z = −1.85, −53.3 / 100k)는 점수상 결핍이 매우 크지만 게시글이 적어 가장 강한 음의 어긋남을 만든다. 노원구는 Safety(z = −1.98, −21.5 / 100k), 연결품질(z = −1.79, −8.1 / 100k), Digital Skills(z = −1.63, −44.3 / 100k)에서 일관되게 목소리가 약하다. 음의 어긋남은 두 가지로 똑같이 설명된다. 실제 결핍이 점수만큼 크지 않거나, 결핍은 있으나 주민이 플랫폼으로 표출하지 않거나. 이러한 사례는 수요의 부재로 읽어서는 안 되며, 해당 자치구·차원 셀에서 플랫폼 가시 신호가 행정 prior보다 약함을 나타낸다.

자치구별로 집계하면 노원구·성동구·구로구는 차원 전반에서 음의 어긋남 쪽으로 기울고, 관악구·중랑구·영등포구는 전반적으로 양의 어긋남 쪽에 선다. 한 지역의 여러 차원이 같은 방향으로 쏠린다는 것은 차원별 결핍의 문제라기보다 그 지역의 표출 성향이 다를 가능성을 시사한다. 다만 이 해석은 데이터가 확증해 주지 못하며, 추가 추론이 필요한 열린 질문으로 남는다. 이 자치구 수준 요약은 추가 질적 검토가 가장 유용한 지점을 식별하는 데 도움을 주는 동시에, 해석을 플랫폼 가시 신호 수준에 머무르게 한다.

Figure 11은 차원 내 표준화 z_shift 값을 공간적으로 매핑하여 지역·차원 괴리가 서울 전체에서 어떻게 분포하는지를 보여 준다. 이 지도는 추정치의 증거 위상을 바꾸지 않으며, 가장 큰 플랫폼-prior 괴리가 어디에 위치하고 공간적으로 군집하는지를 확인하는 데 사용한다.

행정동 매칭과 생활인구 연계는 전처리 품질 통제로 다룬다. 분류된 131,792행 모두를 행정동 후보와 매칭하였고, 150개 자치구·차원 셀 중 147개가 매핑 명세 전반에서 동일한 shift 방향을 유지하였다. 불안정한 3개 셀은 동대문구·서대문구·성북구의 연결품질(CQ) 셀로, 매핑 모호성이 신호를 지배하여 방향을 단정할 수 없다. 이 안정성 수치는 보고된 패턴을 뒷받침하지만 실질적 발견은 아니다.

게시물 분류 자체의 신뢰도를 점검하기 위해, LLM 분류 결과의 일부를 사람이 직접 재검토하는 인간 검증을 수행하였다. 검토 표본에서 LLM이 UMC 관련(Y)으로 분류한 게시물 중 사람이 관련으로 확인한 비율(정밀도)은 84.7%였고, 나머지 15.3%는 사람 검토 결과 UMC와 무관(예: 길거리 사주, 단순 보이스피싱 등)으로 재판정되었다. 사람이 관련으로 확인한 게시물을 기준으로 한 차원 분류 일치도는 완전 일치(부여 차원 집합이 동일) 66.5%, 부분 일치(차원이 하나 이상 겹침) 78.3%였으며, 평균 Jaccard 유사도는 0.723, 주(primary) 차원 기준 Cohen's κ는 0.673이었다. 차원별로는 Safety(F1 0.90), Connection Quality(0.78), Digital Skills(0.73), Affordability(0.68), Available for Use(0.63), Devices(0.51) 순으로 일치도가 높았다. 체계적 편차도 관찰되었다. 사람 검토자는 LLM에 비해 Digital Skills와 Devices를 더 자주 부여하고 Safety와 Connection Quality를 덜 부여하는 경향을 보였다. Table 7은 이 결과를 요약한다. 이 수치는 LLM 분류가 다수 사례에서 사람 판단과 상당히 일치하면서도, 특히 Devices처럼 경계가 모호한 차원에서는 추가 검증이 필요함을 보여 준다. 따라서 본 절의 분류 결과는 구조적이지만 여전히 탐색적인 증거로 다룬다.

## 3.4 분석 결과의 종합

핵심 발견
증거는 한정된 정책 논리를 가리킨다. 서울의 디지털 이용 격차는 주로 개인 취약성으로 설명되며, 자치구 UMC 점수는 타기팅을 위한 맥락적·차원 특정적 조건을 식별한다. 플랫폼 가시 및 Bayesian 신호는 탐색적 경로 증거를 보태지만, 유병률을 추정하거나 인과성을 증명하지는 않는다.

3.1절, 3.2절, 3.3절은 동일한 측정 문제의 서로 다른 부분에 답한다. UMC 지수는 서울 내에서 자치구 조건이 어디에서 차이를 보이는지 보여 주고, HLM은 디지털 이용과 관련된 개인 수준 및 자치구 수준 연관을 분리하며, 플랫폼 가시 텍스트와 Bayesian 결과는 일부 디지털 불이익이 일상 상황에서 어떻게 나타나는지를 보여 준다. 함께 읽으면 어느 한 층도 확증적 증거로 다루지 않으면서 통합적 진단을 뒷받침한다.

주된 실질적 결과는 개인 취약성이 폭넓은 자치구 조건보다 디지털 이용 격차를 더 많이 설명한다는 것이다. HLM 결과는 디지털 이용의 대부분의 변이가 동일 자치구 내 개인들 사이에 존재함을 보여 주며, 가장 강한 패턴은 학력, 연령, 가구 구성 및 이들 간의 교차와 관련된다. 이는 자치구 수준 지표를 개인 디지털 이용의 직접적 원인으로 해석해서는 안 된다는 것을 뜻한다. 이들은 지원의 위치 선정과 설계를 돕는 맥락적 신호이다.

자치구 지수는 UMC의 지리가 차원 특정적이기 때문에 여전히 중요하다. 점수가 낮은 자치구들이 모두 동일한 약점을 공유하는 것은 아니며, 연결성이 비교적 양호한 일부 자치구도 기기, 역량, 안전, 비용 부담 조건에서는 더 약할 수 있다. 이는 장소 민감적 타기팅을 뒷받침한다. 정책은 자치구 프로필을 활용해 취약 주민에게 어디에서 어떻게 도달할지를 결정해야 하며, 자치구 전체를 영구적인 결핍 지역으로 낙인찍기 위해 사용해서는 안 된다.

플랫폼 가시 분석은 이러한 양적 구조에 경로 해석을 추가한다. 일부 이용자는 서비스가 완전히 부재하지 않는 상황에서도 필요한 순간에 정보, 역량, 신뢰할 수 있는 지원, 제도적 경로가 가시화되지 않거나 사용 가능하지 않을 때 디지털 배제를 마주칠 수 있음을 시사한다. 이러한 사례는 예시적이고 탐색적이며, 서베이와 행정 지표가 놓칠 수 있는 기제를 설명하는 데 도움을 주지만, 대표성을 갖는 유병률 추정은 아니다.

EB 수축 율 모형은 작성 시점 시간대 생활인구 노출을 분모로 하여 자치구·차원 수준에서 측정된 UMC prior와 플랫폼 가시 신호를 비교함으로써 이러한 층들을 연결한다. 큰 괴리(per-100k person-years 단위로 보고하고 차원 내 z_shift로 순위를 매김)는 보다 면밀한 조사가 필요한 지점을 표시하며, 특정 자치구나 차원이 객관적으로 더 나쁘다는 것을 증명하지는 않는다. 이 접근은 숨은 호소(점수 양호하나 목소리 강함)와 조용한 결핍(점수 나쁘나 목소리 약함)을 구분하면서도 각각의 증거 위상을 탐색적 수준에 유지한다.

따라서 도출된 정책 논리는 순차적이다. 첫째, 그 특성이 낮은 디지털 이용과 가장 일관되게 연관된 취약 개인과 가구에 대한 지원을 우선한다. 둘째, 자치구 UMC 프로필을 사용해 아웃리치를 타기팅하고 차원별로 개입을 맞춤화한다. 셋째, 관찰된 불이익이 단일 문제가 아니라 흔히 중첩되어 나타나기 때문에 역량, 안전, 비용 부담, 접근 지원을 결합한다. 4장은 이 논리를 정책 권고로 옮긴다.

# 4. 정책 권고

다음 권고는 3.4절에서 종합한 발견으로부터 직접 도출되며, 본 보고서에 제시된 증거에만 기반한다. 권고는 인과적 처방이 아니라 분석과 일관된 정책 방향으로 제시되며, 새로운 정량적 주장을 추가하지 않는다.

## 4.1 개인 취약성 중심 지원

다층 연관성 분석(HLM)은 디지털 이용의 변동 대부분이 자치구 간 차이보다 개인 간 차이와 관련되어 있음을 시사하므로, 지원은 우선 개인 취약성 프로파일을 중심으로 구성되어야 한다. 분석은 낮은 디지털 이용을 낮은 교육 수준, 연속적 기울기로서의 고령, 1인 가구, 그리고 고령과 낮은 교육의 결합과 연관 짓는다. 어느 하나의 인구학적 축을 개별적으로 다루기보다 이렇게 중첩되는 프로파일을 식별하고 도달하는 프로그램이 관찰된 격차의 구조와 가장 일관된다. 플랫폼에서 가시화된 신호는 일부 이용자가 공식적으로 제공되는 서비스를 인지하지 못한다는 점을 추가로 시사하며, 이는 역량 지원을 적극적 아웃리치 및 인지 제고 조치와 결합하는 방향을 뒷받침한다.

## 4.2 장소 민감형 타기팅

자치구 수준 조건은 개인 디지털 이용의 독립적 동인이라기보다 맥락적 타기팅 보조 수단으로 해석하는 것이 가장 적절하다. 인프라 차원은 견고한 평균적 연관성을 보이지 않았으므로, 장소 기반 조치는 개인 중심 지원을 어디에 제공할지 우선순위를 정하고 공간적 군집을 인식하는 데 활용해야 하며, 인프라 단독 투자를 정당화하는 근거로 쓰여서는 안 된다. 잠정적으로 식별된 저점수 자치구들은 약한 차원이 서로 다르므로, 장소 민감형 타기팅은 일률적이기보다 차원별로 특화되어야 하며, Digital Desert 라벨은 고정된 자치구 정체성이 아니라 진단 보조 수단으로 다루어야 한다.

## 4.3 역량·안전·접근성 지원

여러 증거 층위는 단일 프로그램이 아니라 역량(capability), 안전(safety), 접근성(affordability)에 걸쳐 있는 지원을 가리킨다. 역량 지원은 교육 기울기와 플랫폼에서 가시화된 신호 모두에서 드러나는 역량 및 인지 격차에 대응한다. 안전 지원은 탐색적 텍스트 분석에서 표면화된 보안 관련 우려에 대응한다. 접근성 지원은 지수에 포착된 자원 및 부담 조건에 대응한다. 이를 조율된 패키지로 전달하고 공간적 군집이 나타나는 곳에서는 자치구 간 조율을 지원하는 것이, 고립된 개입보다 관찰된 복합 취약성 패턴과 더 일관된다. 이 가운데 어느 방향도 인과적 입증이나 모집단 수준 유병률 추정치로 읽혀서는 안 된다.

| **Core Judgment Principle**: UMC is necessarily about **digital connectivity**. One or more of the following keywords must be addressed as the **primary topic** of the text for it to be UMC-related: |
|  |
| - Internet, Wi-Fi, data, telecommunications, network |
| - **Digital function** usage of digital devices such as smartphones, computers, tablets |
| - **Ability to use / accessibility** of apps, websites, and online services |
| - Telecommunications fees, data charges, internet costs |
| - **Cybersecurity** such as voice phishing, smishing, and online fraud |
|  |
| --- |
|  |
| ## 2. Six Dimensions of UMC — Judgment Criteria |
|  |
| Each text is evaluated against the following six dimensions. A single text may fall under multiple dimensions. |
|  |
| ### 2-1. Connection Quality |
|  |
| - **Core Question**: Can people access high-speed, stable internet connections suitable for their needs and activities? |
| - **Relevant Text Signals**: |
| - Complaints about slow internet/Wi-Fi/data speeds |
| - Experiences of dropped or unstable connections in specific areas |
| - Mentions of quality issues such as video buffering, game lag, or video call disconnections |
| - Comparisons of speed/quality between carriers |
| - Mentions of perceived quality related to 5G/LTE transitions |
| - **Not Applicable**: Simple plan inquiries (→ Affordability), device malfunctions (→ Devices) |
|  |
| ### 2-2. Availability for Use |
|  |
| - **Core Question**: Can people use the internet at the place and time they want, with the frequency and intensity they desire? |
| - **Relevant Text Signals**: |
| - Experiences of no internet access in specific locations (underground, mountains, rural areas, inside buildings, etc.) |
| - Lack of or difficulty accessing public Wi-Fi |
| - Constraints on internet usage time (e.g., data depletion, speed throttling) |
| - Certain services not supported in a region |
| - Absence of offline alternatives due to digital transition (e.g., "오앱으로만 주문 가능", "온라인으로만 신청 가능" — "orders only via app," "applications only online") |
| - **Not Applicable**: Simple store location inquiries, business hours inquiries |
|  |
| ### 2-3. Affordability |
|  |
| - **Core Question**: Are internet access, devices, and data charges affordable and sufficient relative to income? |
| - **Relevant Text Signals**: |
| - Complaints about expensive telecom/internet fees |
| - Requests for affordable plan recommendations |
| - Experiences of limiting usage due to data charge burden |
| - Financial burden of purchasing devices (smartphones, laptops, etc.) |
| - Discussions about budget carriers/low-cost plans (알뜰폰) |
| - **Not Applicable**: General cost-of-living discussions (unrelated to internet or digital devices) |
|  |
| ### 2-4. Devices |
|  |
| - **Core Question**: Can people access appropriate devices to fully leverage digital opportunities? |
| - **Relevant Text Signals**: |
| - Whether people own devices (smartphones, computers, tablets, etc.) or lack them |
| - Inconvenience caused by old or low-performance devices |
| - Experiences of being unable to use digital services due to device malfunction |
| - Experiences of being unable to use services because a specific device is unavailable |
| - Mentions of using shared devices (library PCs, etc.) |
| - **Not Applicable**: Simple device sale/purchase posts, device loss posts, physical exterior repairs (cracked screens, etc.) |
|  |
| ### 2-5. Digital Skills |
|  |
| - **Core Question**: Do people have sufficient skills to leverage digital opportunities and manage potential risks? |
| - **Relevant Text Signals**: |
| - Questions about not knowing how to use apps/websites/digital services |
| - Difficulty using kiosks or unmanned systems |
| - Complaints about complicated online reservation/application/ordering procedures |
| - Inquiries about how to use specific digital features (video upload, file transfer, settings changes, etc.) |
| - Mentions of digital literacy education |
| - Mentions of elderly/senior difficulty with technology use |
| - **Not Applicable**: Professional development/design skill questions, learning offline skills (baking, musical instruments, exercise, foreign languages, etc.), general "I want to learn" expressions |
|  |
| ### 2-6. Safety & Security |
|  |
| - **Core Question**: Can people use secure internet connections, operate safely online, and feel confident about security? |
| - **Relevant Text Signals**: |
| - Experiences/attempts of voice phishing, smishing, or phishing |
| - Anxiety about personal information leaks/theft |
| - Online fraud/impersonation experiences (fraud via internet/apps) |
| - Malware, hacking, ransomware damage |
| - Cyberbullying/online harassment |
| - Exposure to harmful content |
| - Sharing of security tips/prevention methods |
| - **Not Applicable**: Offline crimes (theft, assault, traffic accidents, etc.), Danggeun transaction disputes (bad manners, no-shows, counterfeit goods, etc. — platform etiquette issues, not cybersecurity), offline fraud |
|  |

| ## 3. Mandatory Checklist to Prevent Over-Classification |
|  |
| **Before judging each text, you must ask yourself the following questions. If any answer is "No," it is N.** |
|  |
| ### 3-1. Core Filter Questions (All must be "Yes" for Y or ? to be possible) |
|  |
| 1. **Is the primary topic of this text digital/internet/telecommunications?** — Is the writer's core concern/experience itself about digital connectivity, rather than digital being mentioned merely as background or a tool? |
| 2. **Would this text still make sense if all words like "인터넷" (internet), "와이파이" (Wi-Fi), "데이터" (data), "통신" (telecom), "앱" (app), "온라인" (online), "컴퓨터" (computer), "스마트폰" (smartphone) were removed?** — If yes, digital is a peripheral element, so it is N. |
| 3. **Does solving this problem require a change in digital infrastructure/devices/skills?** — Restaurant recommendations, exercise venues, finding repair services, etc., require non-digital solutions, so they are N. |
|  |
| ### 3-2. Frequently Occurring Misclassification Patterns (Must be judged as N) |

| ## 4. Judgment Logic (Decision Flow) |
|  |
| For each text, follow this sequence: |
|  |
| ``` |
| [Step 1] Read the text → Identify the writer's core concern/experience in one line |
|  |
| [Step 2] Over-classification check (Apply Section 3) |
| ├─ Apply the 3 core filter questions from 3-1 |
| ├─ If it matches a misclassification pattern in 3-2 → Immediately N |
| └─ If it matches "digital appears but N" in 3-3 → Immediately N |
|  |
| [Step 3] UMC Relevance Judgment |
| ├─ Y (Clearly related): The primary topic of the text is a digital connectivity problem/experience |
| ├─ ? (Ambiguous): Digital elements are indirectly related but not the primary topic |
| │   [Criteria for Ambiguity] |
| │   - Digital elements are only indirectly mentioned (e.g., "와이파이 비번 아시는 분?") |
| │   - Device/telecom related but the primary purpose is physical (e.g., "폰 액정 수리 어디가 좋아요?") |
| │   - Mentions a digital service but is closer to a simple information request |
| └─ N (Not related) → Classify as "관련없음" (Not Related), end |
|  |
| [Step 4] UMC Dimension Classification (Only if Y or ?) |
| → Select all applicable dimensions from the 6 (multiple selections allowed) |
| → Must use official dimension names: Connection Quality, Availability for Use, Affordability, Devices, Digital Skills, Safety & Security |
| → Never use numbers (1, 2, 3...) for notation |
|  |
| [Step 5] Problem Summary (Only if Y or ?) |
| → Summarize the specific problem or experience in the text in one sentence in Korean |
| ``` |

| # 판정 에이전트 (Judgment Synthesizer) |
|  |
| ## 역할 |
| 3개 독립 추론 에이전트(귀추적, 전향적, 경로적)의 가설을 입력으로 받아 삼각검증을 수행하고, 최종 분류 코딩을 확정한다. **분류표(absence_typology.json)는 이 에이전트에서만 참조한다.** |
|  |
| ## 입력 |
| 프롬프트로 다음이 주어진다: |
| 1. 게시글 텍스트 + 메타데이터 |
| 2. 3개 에이전트 출력 (abductive, forward, sequential) |

|  |
| ### Step 0: 개별 가설 타당도 평가 |
| 3개 에이전트가 생성한 모든 가설에 대해 타당도를 판정한다. |

| - **confidence**: coding_scheme의 기준에 따라 high/medium/low |

|  |
| ### Step 1: 수렴 분석 |
| 3개 에이전트의 가설이 동일한 결여를 가리키는지 확인한다. |
| - **3/3 수렴**: 모든 에이전트가 동일 결여 식별 → HIGH confidence 부여 |
| - **2/3 수렴**: 2개 에이전트 일치, 1개 상이 → 소수 에이전트의 근거 검토 |
| - **0/3 수렴**: 모두 상이 → Step 2로 이동 |

| 에이전트 간 불일치가 있을 때: |
| - 각 가설의 텍스트 근거(evidence) 강도를 비교한다 |
| - 직접 인용이 더 많은 가설을 우선한다 |
| - "텍스트가 직접 지지하지 않는 정보를 결여 근거로 쓰지 않는다" 원칙 적용 |
|  |
| ### Step 3: 모순 해소 |
| 직접 모순이 있을 때 (예: 귀추적이 "역량 부재", 전향적이 "합리적 선택"): |

| - 어느 쪽이 더 많은 텍스트 단서를 동시에 설명하는지 판정한다 |
| - 해소 서사를 작성한다 (contradiction_notes) |
|  |
| ### Step 4: 맥락 교차검증 |
| 추론된 결여가 해당 맥락에서 개연성이 있는지 확인한다: |
| - 자치구 프로필의 취약 차원과 일치하는가 |
| - 생활인구 구성(고령 비율 등)과 정합하는가 |
| - 게시 시간대가 가설과 부합하는가 |
|  |
| ### Step 5: 비구조적 대안 강제 점검 |
| 최종 판정 전, 반드시 검토한다: |
| - "이 요청은 단순 편의/합리적 선택으로 설명 가능한가?" |
| - "디지털 결여가 아닌 사회적 교류 동기인가?" |
| - 비구조적 대안이 더 설득력 있으면 → confidence를 low로 하향하거나 판단 보류 |
|  |
| ### Step 6: 최종 코딩 (분류표 적용) |
| `config/absence_typology.json`을 참조하여: |
| - absence_type 코드 부여 (A1, A2, B1, B2, C1, C2, D1, E1, F1) |
| - absence_name 한국어명 기록 |
| - umc_dimension 매핑 (`config/umc_dimensions.json` 참조) |
| - confidence 확정 (high/medium/low) |
| - 확신도가 판단 보류 수준이면 전체 게시글을 suspend 처리 |
| - layer 확정 (individual/local/institutional) |
| - surface_deficit (표면적 결핍) 기술 |

|  |

| - 경로 단절 지점 + 의존 패턴이 기존 범주와 유사한가 |
| - **absorb**: 기존 범주에 흡수 (구조 동일) |
| - **slip**: 기존 범주와 1차원 어긋남 → 범주 정의 미세 수정 필요 |
| - **bifurcate**: 기존 범주에 대응 없음 → 신규 범주 후보 |
| - **해당없음**: suspend 처리 |
|  |
| ## 출력 형식 |

| - A1-F1 등 분류 코드 사용 금지 |
|  |
| ## 추론 절차 |
|  |
| ### Step 1: 놀라운 사실(C) 식별 |
| 이 사람이 **공식 경로나 검색 대신** 동네 커뮤니티에 이 요청을 올린 것이 놀라운 사실이다. |

| "만약 조건 A가 성립한다면, 이 사람이 커뮤니티에 요청하는 행위(C)가 기대된다" |

| "무엇이 있었더라면 이 요청은 발생하지 않았을까?" |
| - **개인 수준**: "이 사람이 ~할 수 있었더라면", "이 사람에게 어떤 도움이 있었더라면" |
| - **환경 수준**: "이 동네에 ~가 있었더라면", "제도가 ~했더라면" |
| - 반사실을 가설로 번역한다. 개인 수준에서 멈추지 않는다. |
|  |

| 비구조적 대안도 가설로 생성한다: |
| - 단순 편의 (검색보다 커뮤니티가 빠르다) |
| - 합리적 선택 (비용 최소화) |
| - 사회적 동기 (이웃과 교류하고 싶다) |
| 이 대안이 텍스트로부터 추론 가능하면 별도 가설로 출력한다. |
|  |
| 가설의 타당도(confidence, layer) 판정은 이 에이전트의 범위 밖이다. |
| 판정 에이전트가 수행한다. |
|  |

| # 전향적 추론 에이전트 (Forward Reasoner) |
|  |
| ## 추론 방향 |
| **조건 → 예측 → 대조**: 게시글의 상황 조건으로부터 예상되는 디지털 결여 발현을 추론하고, 실제 게시글 내용과 대조한다. |
|  |
| ## 입력 |
| 프롬프트로 다음이 주어진다: |
| 1. 게시글 텍스트 (post_id, gu, dong, created_at, subject, title, content, 작성자 ID) |
| 2. 이론 체계 (`config/coding_scheme.json`) |
|  |
| 맥락 블록(생활인구, 자치구 프로필)은 이 에이전트에 전달하지 않는다. |
| 추론은 게시글 텍스트와 메타데이터만을 근거로 수행한다. |
|  |
| ## 금지 |
| - `config/absence_typology.json` 참조 금지 |
| - `knowledge/categories/active.json` 참조 금지 |
| - 다른 추론 에이전트의 출력 참조 금지 |
| - A1-F1 등 분류 코드 사용 금지 |
|  |
| ## 추론 절차 |
|  |
| ### Step 1: 상황 특성화 |
| 게시글 텍스트와 메타데이터에서 관찰 가능한 지표만으로 작성자의 상황을 특성화한다: |
| - **공간**: 메타데이터의 gu·dong에서 읽을 수 있는 정보만 사용 |
| - **시간**: 게시 시각(created_at)에서 도출 가능한 요일·시간대·시기 |
| - **과업**: 게시글에 드러난 과업의 성격 |
| - **화자 단서**: 텍스트에서 드러나는 화자의 특성 (어투, 언급된 상황 등), 작성자 ID 참조 |
|  |
| ### Step 2: 결여 발현 예측 |
| "이 상황의 사람이라면 어떤 디지털 연결성 어려움을 겪을 것으로 예상하는가?" |
| - coding_scheme의 breakpoint_types를 참조하여 예측한다 |
| - 예측은 **상황 조건에서 논리적으로 도출**되어야 한다 |
| - 텍스트가 지지하는 만큼 예측을 생성한다 (개수 제한 없음) |
|  |
| ### Step 3: 게시글과 대조 |
| 각 예측을 게시글 실제 내용과 비교한다: |
| - **일치(match)**: 예측한 결여가 게시글에서 실제로 표현됨 → 가설로 승격 |
| - **불일치(mismatch)**: 예측했으나 표현되지 않음 → 기록 (판정 에이전트에 유용) |
| - **미예측 발견**: 예측하지 못한 결여가 게시글에서 발견됨 → 별도 기록 |
|  |
| ### Step 4: 가설 생성 |
| 대조 결과(match, mismatch, unexpected 모두)에 대해: |
| - 기저 결여(underlying deficit)를 가설로 구성한다 |
| - 텍스트 근거를 명시한다 |
|  |
| 가설의 타당도(confidence, layer) 판정은 이 에이전트의 범위 밖이다. |
| 판정 에이전트가 수행한다. |
|  |
| ## 출력 형식 |
| 게시글당 **가설 1~2개**만 출력한다. 가장 텍스트 대조가 강한(match) 가설을 선택한다. |
| 추가 설명이나 서문을 붙이지 않는다. |
| 메타데이터(gu, dong, author_id, created_at)는 **생략**한다 — 입력에 이미 존재. |
| 각 필드는 **1문장 이내**. |
|  |
| ```json |
| [ |
| { |
| "id": "FWD-{post_id}-1", |
| "post_id": "", |
| "agent": "forward", |
| "situation": "공간·시간·과업·화자 특성 요약 1문장", |
| "prediction": "예측된 결여 발현 1문장", |
| "match": "match¦mismatch¦unexpected", |
| "deficit": "기저 결여 1문장", |
| "evidence": ["직접 인용 1~2개"], |
| "breakpoint": "coding_scheme의 breakpoint_type 중 하나" |
| } |
| ] |
| ``` |

|  |
| # 경로적 추론 에이전트 (Sequential Reasoner) |
|  |
| ## 추론 방향 |
| **경로 재구성**: 게시글 작성자가 문제를 인식한 시점부터 커뮤니티에 도움을 요청하기까지의 경로를 순차적으로 재구성하고, 경로 이탈 지점을 식별한다. |
|  |
| ## 입력 |
| 프롬프트로 다음이 주어진다: |
| 1. 게시글 텍스트 (post_id, gu, dong, created_at, subject, title, content) |
| 2. 이론 체계 (`config/coding_scheme.json`) |
|  |
| 맥락 블록(생활인구, 자치구 프로필)은 이 에이전트에 전달하지 않는다. |
| 추론은 게시글 텍스트와 메타데이터만을 근거로 수행한다. |
|  |
| ## 금지 |
| - `config/absence_typology.json` 참조 금지 |
| - `knowledge/categories/active.json` 참조 금지 |
| - 다른 추론 에이전트의 출력 참조 금지 |
| - A1-F1 등 분류 코드 사용 금지 |
|  |
| ## 추론 절차 |
|  |
| ### Step 1: 경로 재구성 |
| 게시글 텍스트에서 작성자의 행위 순서를 재구성한다: |
| - 무엇을 하려 했는가 (목표) |
| - 무엇을 먼저 시도했는가 (시도된 경로) |
| - 어디서 막혔는가 (이탈 지점) |
| - 왜 커뮤니티에 왔는가 (최종 선택) |
|  |
| coding_scheme의 `pathway_components`를 참조하되, 텍스트에 드러난 순서를 우선한다. |
| 텍스트에 명시되지 않은 단계는 "(추정)"으로 표시한다. |
|  |
| ### Step 2: 정상 경로 대비 |
| 각 단계에서 "정상적" 다음 단계가 무엇이었을지 식별한다: |
| - 공식 디지털 채널 (웹사이트, 앱, 고객센터) |
| - 검색 엔진 |
| - 제조사/서비스 제공자 직접 연락 |
| - 오프라인 공공 서비스 |
| - 행정적 도움 |
|  |
| ### Step 3: 경로 이탈 지점(breakpoint) 식별 |
| 정상 경로에서 벗어난 지점을 정확히 지정한다: |
| - **공식 채널 시도 → 실패 → 커뮤니티**: 채널이 존재하나 기능하지 않음 |
| - **공식 채널 인지 부재 → 커뮤니티 직행**: 경로 자체를 모름 |
| - **공식 채널 접근 불가 → 커뮤니티**: 시간/비용/물리적 장벽 |
|  |
| 이 구별이 핵심이다. 텍스트에서 이전 시도의 흔적을 찾는다: |
| - "~에 전화했는데", "~에서 안 된다고" → 시도 후 실패 |
| - 이전 시도 언급 없음 → 인지 부재 또는 직행 |
|  |
| ### Step 4: 이탈 지점별 가설 생성 |
| 각 breakpoint에서 무엇이 부재했는지 가설을 구성한다: |
| - coding_scheme의 `breakpoint_types`를 참조한다 |
| - 텍스트 근거를 명시한다 |
| - 연쇄 실패(cascade) 여부를 표기한다 |
|  |
| 가설의 타당도(confidence, layer) 판정은 이 에이전트의 범위 밖이다. |
| 판정 에이전트가 수행한다. |
|  |
| ## 출력 형식 |
| 게시글당 **가설 1~2개**만 출력한다. 가장 텍스트 근거가 명확한 이탈 지점을 선택한다. |
| 추가 설명이나 서문을 붙이지 않는다. |
| 메타데이터(gu, dong, author_id, created_at)는 **생략**한다 — 입력에 이미 존재. |
| 각 필드는 **1문장 이내**. |
|  |
| ```json |
| [ |
| { |
| "id": "SEQ-{post_id}-1", |
| "post_id": "", |
| "agent": "sequential", |
| "goal": "작성자의 원래 목표 1문장", |
| "tried": "시도된 경로 1문장 또는 '언급 없음'", |
| "derail": "tried→failed¦no_awareness¦access_blocked", |
| "absent": "이탈 지점에서 부재한 것 1문장", |
| "cascade": true/false, |
| "evidence": ["직접 인용 1~2개"], |
| "breakpoint": "coding_scheme의 breakpoint_type 중 하나" |
| } |
| ] |
| ``` |
