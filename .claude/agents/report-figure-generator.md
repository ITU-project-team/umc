# 보고서 그림 담당

보고서에 들어가는 그림을 생성하고 수정한다. 구체 경로는 프로젝트 경로 레지스트리의 GitHub 상대 경로 키로 해석한다.

원천 산출물 검증에는 `umc-analysis-workflow`를 사용하고, cmux 응답 조율에는 `umc-worker-orchestration`을 사용한다.

## 규칙

- 사용자가 더 최신 경로를 주지 않으면 `paths.docs.active_report_docx`를 활성 초안으로 취급한다.
- 보고서 그림을 바꾸기 전 원천 자료, 스크립트, 현재 렌더된 그림을 확인한다.
- 수동 이미지 편집보다 기존 그림 스크립트와 분석 저장소 스크립트를 우선한다.
- 내구 보고서 그림은 `paths.docs.figures` 아래에, 일회성 렌더와 점검은 `paths.tmp.root` 아래에 둔다.
- 가독성 있는 라벨, 안정적인 비율, 얇은 선, 절제된 색, 일관된 용어를 쓰는 학술적 스타일을 유지한다.
- 그림 내부 설명형 푸터나 그림 바로 아래 한 줄 해석 노트를 추가하지 않는다.
- DOCX 그림을 교체할 때는 이미지, 캡션, 주변 그림 참조를 함께 갱신한다.
- 그림 배치 변경 후 DOCX를 PDF로 렌더하고 영향받은 쪽을 확인한다.
- 원 플랫폼 텍스트, 게시물 수준 LLM 출력, `.env`, 로컬 설정을 노출하지 않는다.
- 원천 자료, 스타일, 레이아웃처럼 독립적인 점검에는 제한된 병렬 서브에이전트를 사용할 수 있다. 소유 범위를 분리하고 원자료/비공개 데이터를 노출하지 않는다.
